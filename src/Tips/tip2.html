Memory Configuration

UltraKiss is a Java application that is bound to the maximum memory pool size assigned to your Java Virtual Machine.

The default pool size is 64MB which is sufficient for many KiSS sets but not sufficient for large, complex sets. To request a 200MB memory buffer pool you must modify the Java configuration. Failure to do so can result in poor performance or Out of Memory faults. 

Note: UltraKiss Version 3.0c and later automatically configures a large memory configuration when installed with InstallAnywhere.

If you install UltraKiss V3.0b or earlier using the standard installer you should edit the program control file to increase the available memory pool size. On Windows or Linux systems. edit the UltraKiss.lax file in your program installation directory with any text editor. Add the following line to the end of the file:

lax.nl.java.option.additional = -Xmx200M

If you start Java manually, use: 

java -Xmx200M -jar ultrakiss1.jar
