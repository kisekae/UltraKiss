removeItem() 

Syntax:     removeItem (item,component)

This action removes the specified item from a list component or combobox component.. The item may be a literal string name enclosed in double quotation marks or a variable whose value can be converted as a string. The component may be a literal name of a list or combobox cel or a variable whose value is a string name of a list or combobox. The action command name 'removeItem' is case sensitive.

This command fails silently if the specified component does not maintain items or if the specified item is not contained in the component. Note that item comparisons are case sensitive.

Components are declared as cels within a KiSS configuration as shown in the example below.  Component cel identifiers use specific extensions such as .button, .list, or .textarea to identify the component type.  

#1.9999 List-1.List           : 0     ;%size[146,207] %offset[12,121] %attributes[wrap,vsb]
#1.9999 TextArea-1.TextArea   : 0     ;%size[269,49] %offset[227,176] %attributes[wrap]
#1.9999 Button1.Button        : 0     ;%size[135,25] %offset[325,267] %attributes[text="Press Me"]

The example above defines a List component with vertical scroll bars, a TextArea component that performs word-wrap, and a Button.  The size and location of these components within the object are specified with the %size[] and %offset[] tags.  Component attributes are specified with the %attributes[] tag.  

FKiSS press() and release() events can be attached to component objects.  These events fire when the user interacts with a component to either press a button, change the state of a checkbox, or select an item from a list or combo box.  

Components are either Input components, or non-Input components.  Input components, such as a multi-line Text Area or a single line Text Field are capable of accepting user input or otherwise manipulated through user selection.  Non-input components, such as a standard Button or a Text Area that has the read attribute assigned do not accept input and are passive.  Passive components are similar to a KiSS image cel.  These components can be drawn, layered, and positioned on the screen along with other KiSS images.  Conversely, active input components intercept mouse events and cannot be directly manipulated with mouse movements.  Input components are drawn on top of all other images in the KiSS set and can extend beyond the playfield boundaries.

Related actions: addItem(), removeAll(), getItemCount(), getSelectedItem(), setSelectedItem()

 

Compatibility:

UltraKiss Extension removeItem ("apple","list1.list") removes the specified item string from the named component

 

Note: The UltraKiss removeItem() command is viewer specific and not compatible with other viewers.

 

Example: 

#3.999 textfield1.textfield   : 0 ;%size[318,26]     %attributes[text="Item Description"] 
#3.999 list1.list             : 0 ;%size[175,177]    %attributes[vsb] 

;@press("button1.cel") ; 
;@ getText(S,"textfield1.textfield") ; 
;@ removeItem("list1.list",S) ; 

;@press("button2.cel") ;
;@ let(List,"combobox1.combobox")
;@ getText(S,"textfield1.textfield") ; 
;@ removeItem(List,S) ;
