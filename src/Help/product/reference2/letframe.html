letframe()

Syntax:     letframe (variable,target)

The letframe() statement sets the variable to the frame number that is currently visible in the target, or to -1 if no frame has been set for the target. In KiSS, a frame is a collection of cels in a cel group, or one cel in an object group, or a separate frame within an image cel such as a GIF image. When a frame is set for the group all images within the frame are made visible, or mapped, and all other images within the group are unmapped.

The target can be a cel group or an object group or a cel name specifier.

FKiSS 4 only considered the notion of frames within cel groups. FKiSS 5 extends the frame concept to object groups and animated image cels.

Note that it is possible for frames to be set for a group where no such frame has been defined. In this case all images in the group are unmapped and the group as a whole is unmapped. The value returned by letframe() for a completely unmapped group is -1.

 

Cel Group frames

For cel groups, individual cels can be identified as being in a sub-collection of cels, or frame. This sub-collection can be made visible or mapped with a setframe() command. When a frame is mapped all other frames in the group are unmapped.

For cel groups, f rames are specified using the ! tag on the cel definition line in the KiSS configuration file.

Tag Syntax:             ; !<name> [: n1 n2 n3]

The celgroup tag is the cel group name starting with an exclamation mark, and optionally a frame definition which is a colon followed by frame numbers. As the example below shows, one cel instance can be assigned to many different cel groups. The example shows that cels 'three.cel' and 'four.cel' both participate in the !Number and !Card groups, but individually in the !Three and !Four groups. Also note that within the !Card group, both image cels participate within frame number 1, but only the 'three.cel' image participates in frame number 2.

Example:

#10.100 three.cel : 0 ; !Number :3 
;                       !Card :1 2
;                       !Three
#10.100 four.cel  : 0 ; !Number :4 
;                       !Card :1 
;                       !Four

Object Group frames

For object groups, each cel within the group is automatically identified by its sequence within the KiSS configuration file. Object group frame numbers begin from zero. Object group frames simplify the selection and visibility mapping of an individual cel within an object.

#10.100 one.cel   : 0 ; this is implicit frame 0
#10.100 two.cel   : 0 ; this is implicit frame 1
#10.100 three.cel : 0 ; this is implicit frame 2
#10.100 four.cel  : 0 ; this is implicit frame 3

 

Image Cel frames

For image cels that support multiple internal frames, the letframe() and setframe() commands can be used to selectively identify and display a specific frame in an animated image. UltraKiss currently provides frame support for GIF image cels.

 

Related events: ifmapped(), ifnotmapped()

Related actions: setframe(), map(), unmap(), letmapped()

 

 

Compatibility:

FKiSS 5   letframe(N,"counter.gif")  Variable N is set to the current visible frame in "counter.gif"
FKiSS 5 letframe(N,#1)  Variable N is set to the current visible frame in object #1

 

UltraKiss Options:

     
VariableCase variable names are case sensitive default value "false" (Compatibility option)

 

 Example: 

;@letfix(N,"shirt.cel") ; N contains the fix value for the specified cel
;@letfix(N,#3) ; N contains the largest fix value for all cels in the object #3
