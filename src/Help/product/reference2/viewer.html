viewer()

Syntax:     viewer (function,operand1,operand2,...)

This command invokes specialized functions specific to the viewer program. The function is specified as a string literal or variable and the operands can be literal values or variables that represent function specific values. 

Viewer Functions

"reset" This causes a viewer reset, similar to invoking the View-Reset menu function.
"restart" This causes a viewer restart, similar to invoking the View-Restart menu function.
"getoption" This gets the value of an UltraKiss viewer options.  The argument arg1 is the option name and arg 2 is a variable for the returned value. Option names are shown below.  
"setoption" This sets or clears UltraKiss viewer options.  The argument arg1 is the option name and arg 2 is the required value. Option names are shown below.  
"select" Shows a configuration selection dialog, similar to invoking the View-Select menu function.
"open" Opens the file specified in arg1.  If the file is a CNF configuration file a new configuration will be loaded, otherwise the appropriate tool is accessed. This command opens files from the default directory.
"openarchive" Opens the archive file specified in arg1.  If the file contains only one CNF configuration file this new configuration will be loaded. If it contains more than one CNF element a selection dialog will be opened to choose the correct configuration file. If no CNF element is found nothing is opened.
"openURL" Opens the archive file specified in arg1.  The file specified must be a valid URL with an http:// or https:// protocol. The URL file will be downloaded into cache. If the file contains more than one CNF element a selection dialog will be opened to choose the correct configuration file. If no CNF element is found nothing is opened.
"openURL" Opens the archive file specified in arg1.  The file specified must be a valid URL with an http:// or https:// protocol. The URL file will be downloaded into cache. If the file contains more than one CNF element a selection dialog will be opened to choose the correct configuration file. If no CNF element is found nothing is opened.
"view" Opens a text editor to display the specified KiSS file entity specified in arg1.  The file entity is a file object previously opened with an 'open()' command.
"print" Opens the print subsystem to print the specified file entity specified in arg1.  The file entity is any file object previously opened with an 'open()' command. 
"exit" Exits the program and terminates the UltraKiss viewer.  
"setmenu" Replaces the UltraKiss menu bar with a user specified menu.  For menu specifications see the example.
"seticon" Replaces the UltraKiss icon image with a user specified image.  The argument arg1 should specify a cel name that is to be used for the icon image. 
"settitle" Sets the UltraKiss main window title.  The argument arg1 should specify the text string that is to be used for the window title.
"menu" Invokes an UltraKiss menu command.  The argument arg1 specifies the required menu item as specified in the Viewer Menu Items list below.

Viewer SetMenu Example

Viewer "setmenu" arguments arg1, arg2, arg3, ... and so on specify the user menu items in pairs, where the first argument is the menu name and the second argument refers to an object group that contains a set of MenuItem or CheckBoxMenuItem components for the menu.   The following command would create a three item menu bar:

viewer("menu","File",#20,"Options",#21,"Help",#22) 

The object definitions for objects #20, #21, and #22 should contain MenuItem or CheckBoxMenuItem components as shown below:

#20  filenew.MenuItem                ; %attributes[text="New"]
#20  fileopen.MenuItem               ; %attributes[text="Open"]
#20  fileclose.MenuItem              ; %attributes[text="Close"]

#21  optionshow.CheckBoxMenuItem     ; %attributes[text="Show Colors"]
#21  optionhide.CheckBoxMenuItem     ; %attributes[text="Hide Background"]

#22  filehelp.MenuItem               ; %attributes[text="Help"]

 

Related actions: environment()

 

Compatibility:

UltraKiss Extension viewer("reset") or viewer("menu","activecnf") invoke viewer specific functions

 

Note: The UltraKiss viewer() command is viewer specific and not compatible with other viewers.

 

Example Code (FKiSS5): 

;@press(#79)                        ; 
;@ viewer("menu","activecnf")       ; show source code

;@press(#80)                        ; 
;@ viewer("option","sound","false") ; turn all sound off

;@press(#81)                        ; 
;@ viewer("reset")                  ; invokes a 'reset' menu command
;@ viewer("menu","reset")           ; this is equivalent to the previous command
