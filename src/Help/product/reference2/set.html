set() 

Syntax:     set (integer)

This event is triggered when the active viewer page changes to the specified page. All KiSS pages are numbered and the first page begins from zero.

The maximum number of pages in a KiSS set is established from the number of $ page configuration lines in the set configuration file. Note that the first page displayed may not always be page 0. The first page displayed is the first page with a visible cel, and this may not always be page 0. 

Many viewers will not support more that 10 pages although there is no restriction as to the number of pages in a set.

The set() event is recognized when the active page on display changes through a mouse menu selection or through FKiSS changeset() commands. KiSS pages are also associated with a default palette group and a page change can also trigger a col() event.

A generic set(*) event is defined to fire on any page change. Note that you can specify both a generic set(*) event and specific set() events. Specific set() events will always supercede a generic set(*) event. If a specific set() event exists then the generic event will not fire on a change to the specified page. 

More than one set() event can be specified for a specific page and each event will fire in the sequence in which it was defined. Set() events are immediate and are not queued for later processing. These events fire before color change col() events.

Set() events will fire on changeset() action commands used in initialize() events.


Related events:  col()
Related actions: changecol(), letmaxcolor(), changeset(), letmaxpage()

 

Compatibility:

FKiSS 1 allows integer page numbers. set (4) Triggered when page 4 becomes active
FKiSS 4 allows generic set(*) events.   set (*)   Triggered when any page becomes active
FKiSS 5 allows an unlimited number of pages  set (99)   Triggered when page 99 becomes active

 

UltraKiss Options:

MaxPageSet maximum pages displayed on the toolbar default value 10 (KiSS option)
Page Up keyboard command to show prior page
Page Dn keyboard command to show next page

;[Palette Section]

%smoon04a.kcf 
%smoon07.kcf

#17 smoon04.cel *1 : 2 ;
#16 smoon02.cel *0 : 1 ;

;@ notify("Your viewer allows changeset() to empty pages.") ;
