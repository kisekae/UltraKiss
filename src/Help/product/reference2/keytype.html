keytype() 

Syntax:     keytype (keyliteral)

This event is triggered by releasing the key with the specified value. The keytype() event is case sensitive, therefore keytype("a") and keytype("A") are two different events. The keytype() event applies to all printable keys on the keyboard. The key literal should be enclosed within double quotation marks.

FKiSS 5 introduced the keytype() event to enable case sensitive input of keyboard entries. This event is paired with the letkeycase() and letkeymodifier() action commands to pick up the actual key combination pressed and released. With FKiSS 5 string processing functions and the letkeystring() action command the capability now exists to pick up keyboard input and process it in a case sensitive and context sensitive way. 

More than one keytype() event can be specified for a specific key. If sequential event processing is enforced each event will fire in the sequence in which it was defined, otherwise they are queued for concurrent processing. 

Note that there is no defined order for event processing in KiSS. Single threaded viewers that do not support concurrent events will typically process the keytype() event after the keypress() and before the keyrelease() events. However, for multithreaded viewers that support concurrent event processing there is no guarantee that the keypress() event will complete before the keytype() event begins. No assumptions should be made for the sequence in which events are processed.

Related events: keypress(), keyrelease()
Related actions: letkey(), letkeymodifier(), letkeymap(), leykeychar(), letkeycode(), letkeystring()

 

Compatibility:

FKiSS 5 allows all keys.  keytype ("a")  or  keytype ("Space") Triggered when the key is released on the keyboard.

 

UltraKiss Options:

KeyCase enable or disable key case recognition default value "true" (FKiSS option)
RetainKey  active key press is required for letkey() and letkeymap() actions  default value "true" (Compatibility option)

;@ notify("A key typed") ;
