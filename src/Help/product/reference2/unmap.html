unmap() 

Syntax:     map (target)

This action clears the mapping or visibility of the target. The target may be a literal name of an image file or a cel group name or an object group identifier.

For object groups or cel groups, each image cel in the group is hidden.

FKiSS adjustment of cel visibility through un map() commands can trigger collision events if the viewer supports visibility collisions. For example, an image collide() event could fire when the cel becomes visible, or an apart() event could fire when the cel was hidden.


Related actions: altmap(), map(), setframe(), letframe(), ghost()

 

Compatibility:

FKiSS 1 allows cel literal names or objects. unmap ("cel1.cel") or unmap(#1) Clears the mapping of the specified cel, or all cels in the object group.
FKiSS 3 allows variables for objects.   let(X,10) unmap (X)  Clears the mapping of the all cels in the object group specified by variable X.
FKiSS 4 allows cel group names.   unmap (!Table)  Clears the mapping of all cels in the cel group.
FKiSS 5 allows variables for all names let(Cel,"a.cel") unmap (Cel)  Clears the mapping of the specified cel, or all cels in the object group or cel group.

 

UltraKiss Options:

AllAmbiguous apply unmap to all ambiguous cels default value "false" (Compatibility option)
MapCollide apply collision events on visibility changes default value "false" (Compatibility option)

 

 Example: 

#1.999 button.cel : 0 ;      
#3.999 fk4b3a.cel : 0 ;      normal image
;@press(#1)   map(#3)  ; 
;@release(#1) unmap(#3) ;
