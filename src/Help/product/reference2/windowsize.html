windowsize() 

Syntax:     windowsize (width, height)

A windowsize() command is used to create a window into the KiSS set that is smaller than the playfield size. Windows, then, are subsets of the playfield. Windows into the playfield are moved using viewport() commands.

The windowsize width and height are integer parameters that are relative width and height adjustments, in pixels, from the current window size. Negative values make the window smaller and positive values make the window larger. Note that some viewers such as DirectKiss have mistakenly interpreted the width and height parameters as absolute dimensions, which is not correct.

Whenever a windowsize() command is executed the viewport is centered on the new window.

The window size cannot be increased beyond the playfield boundaries. The window size can be decreased to nothing.

 

Related actions: viewport()

 

Compatibility:

FKiSS 1 allows literal names or objects. windowsize (-100,-100) Make a window 100 pixels smaller in width and height
FKiSS 3 allows variables for objects.   let(X,10) windowsize (X,0)  Make the window 10 pixels larger in width

 

 Example: 

;@windowsize(-100,-100)    ; Make a window 100 pixels smaller in width and height
;@viewport(-10,22)         ; Move the window 10 pixels left and 22 pixels down
