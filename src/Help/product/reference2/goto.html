goto() 

Syntax:     goto (labelname)

The goto() command transfers code execution to the specified label module. The next FKiSS statement executed is the first statement in the label module. There is no return to the calling event or the label module with the goto() statement, all code statement following the goto() are not processed.

The goto() statement and label modules were introduced in FKiSS 3 as a means to structure FKiSS code so that common functions could be isolated and called as required.

The label identifier can be an integer or a name. Label identifiers must be unique within the set of all label modules.

The goto() command will fail silently if the specified label module does not exist.

 

FKiSS 5 Syntax:     goto (labelname[, argument1, argument2, ...])

FKiSS 5 extended the notion of labels to allow for parameter specification and return values. Previous FKiSS versions were incapable of implementing recursive label modules or functions. FKiSS 5 implements local label variables and this provides the ability to support recursion and function return values. Local variables are label specific and eliminate the requirement for the programmer to manage global variables.

The optional argument list specifies a set of variable names that must be referenced as local variables within the label module.

Local variable names are prefixed with an '@' sign. Thus, the variable '@N' is a local variable declared and referenced within the module and is distinct from any global variable 'N'. Calling the label module recursively creates new instances of all local variables distinct from any other local variable of the same name.

There can be no return from a label module invoked through a goto() statement. The following example shows how to specify label parameters for a module invoked through a goto() statement.

 

Related actions: label, gosubrandom(), gosub(), gotorandom()

 Example:

;@ let(I,3) let(J,2)
;@ debug("about to multiply")
;@ goto(Multiply)
;@ debug("back from multiply") ; this is never executed

;@label(Multiply)
;@ debug("doing a multiply") 
;@ mul(X,I,J)

FKiSS 5 Example:

;@EventHandler
;@begin() 
;@ goto(Add,5,3)               ; 5 and 3 are arguments to the Add label routine
;@ notify("This should not be displayed") ; this statement will never be executed

;@label(Add,@N1,@N2)           ; local variable @N1 and @N2 are the parameters
;@ add(X,@N1,@N2)              ; X is a global variable
;@ notify("Sum is ",X)         ; show the sum
