getNextSelectedIndex() 

Syntax:     getNextSelectedIndex(variable,component)

This is a component command to retrieve the next index number of a list component or combobox component that supports multiple selections and has a selected item. The component may be a literal name of a list or combobox, or a variable that contains the name of a list component or combobox component. Note that the action command name 'getNextSelectedIndex' is case sensitive and must be specified as shown.

This command sets the specified variable to the integer value of the next selected item index in the list or combobox for those components that have the 'multiple' selection attribute set. The next selected item always begins with the first selected item if no previous getSelectedIndex() or getNextSelectedIndex() command has been issued for the component. If the last selected item in the list has been returned through a prior getNextSelectedIndex() command then the command will return the first selected item index. If there is no currently selected item in the list or if the component is not a list or combobox the variable is set to -1. Note that indexing of list items begins at 0.

Components are declared as cels within a KiSS configuration as shown in the example below.  Component cel identifiers use specific extensions such as .button, .list, or .textarea to identify the component type.  

#1.9999 List-1.List           : 0     ;%size[146,207] %offset[12,121] %attributes[wrap,vsb]
#1.9999 TextArea-1.TextArea   : 0     ;%size[269,49] %offset[227,176] %attributes[wrap]
#1.9999 Button1.Button        : 0     ;%size[135,25] %offset[325,267] %attributes[text="Press Me"]

The example above defines a List component with vertical scroll bars, a TextArea component that performs word-wrap, and a Button.  The size and location of these components within the object are specified with the %size[] and %offset[] tags.  Component attributes are specified with the %attributes[] tag.  

 

Related actions: getSelectedIndex(), setSelectedIndex(), setSelectedValue(), setSelectedItem()

 

Compatibility:

UltraKiss Extension getNextSelectedIndex (N,"list1.list") Sets the variable N to the index of the next selected item in the list

 

Note: The UltraKiss getNextSelectedIndex() command is viewer specific and not compatible with other viewers.

 

Example: 

#8.999 button1.button         : 0 ;%size[318,26]     %attributes[text="Press Me!"] 
#3.999 list1.list             : 0 ;%size[175,177]    %attributes[vsb]
