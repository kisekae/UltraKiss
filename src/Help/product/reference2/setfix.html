setfix()

Syntax:     setfix (target,integer)

The setfix() statement sets the lock or fix value associated with the cel or object to the specified integer value. The integer value can be specified as a literal value or a variable. Fix values should be non-negative numbers.

The target can be a cel name or an object group specifier. FKiSS 3 allowed only object groups for the setfix() statement, FKiSS 5 allows cel names.

The setfix() statement can trigger an unfix() event if the object or cel fix level is set to 0. Many viewers will interrupt the current execution to fire the unfix event immediately although there is no standard definition as to when this should occur. UltraKiss maintains compatibility options to control when such events are fired.

Some viewers will fire unfix() events on unmapped objects. UltraKiss maintains a compatibility option to enable or disable unfix() events based upon object or cel visibility.

If the user is currently moving the target object with the mouse and an external alarm performs a setfix() on the object where the fix value is greater than zero, then the user will lose control of the object and a fixdrop() event will fire. Because the object is now fixed and so should not have moved it will snap back to its original position when first picked up by the mouse. Other viewers may not implement this behaviour and can drop the object at a different location. This can lead to viewer incompatibility if inappropriate collision events fire when the object is dropped.

It is the object that is fixed and the fix count for an object group is usually defined as the maximum value of all lock values for all cels in the object, although some viewers may define this differently. If the fix count for a cel is changed then the fix count of the cel object is also changed. This can result in unfix() events being fired for both the cel and the object.

 

Related events: unfix(), fixdrop()

Related actions: letfix()

 

 

Compatibility:

FKiSS 5   letfix(N,"shirt.cel")  Variable N is set to the declared lock value for "shirt.cel"

 

UltraKiss Options:

     
VariableCase variable names are case sensitive default value "false" (Compatibility option)

 

 Example: 

;@letfix(N,"shirt.cel") ; N contains the fix value for the specified cel
;@letfix(N,#3) ; N contains the largest fix value for all cels in the object #3
