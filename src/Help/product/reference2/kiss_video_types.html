KiSS Video Types

When KiSS was invented in the early 1990's, computer video did not exist. Some early KiSS artists did try to implement movie type capabilities by animating series of images, but techniques today would incorporate movie format cels or animated graphic images in KiSS sets. KiSS viewers have not progressed to this point and UltraKiss is the only viewer to provide the artist with ways to experiment with such capabilities.

Movies are video cels defined with an AVI or MPG extension for a supported video file type. Such cels are essentially indistinguishable from standard image cels, except that they may include playback controls and other features to manage presentation.

The ability to play movies is an experimental feature of a KiSS viewer. Music and sound and movies are all separate entities and playback of movie files is distinct from sound files.

Movie files should be played concurrently. The movie("") command with no file name specified will stop all movie playback instantly. 

 

Example:

 
 ;[Cel Section]
#2.999 button2.button : 0 ;%size[100,50]  %attributes[text="Stop"] 
#2.999 button1.button : 0 ;%size[100,50]  %attributes[text="Play"] 
#1 FOX.AVI            : 0 ;%size[320,240] %attributes[repeat] 

;[FKiSS Section]
;@EventHandler

;@initialize() ; 
;@ unmap("button2.button") ; 

;@press("button2.button") ; 
;@ movie("") ; 
;@ altmap(#2) ; 

;@press("button1.button") ; 
;@ movie("fox.avi",-1) ; 
;@ altmap(#2) ; 

 

Compatibility:

UltraKiss is the only viewer to experiment with video cels and animated graphic cels.

UltraKiss introduced the movie() action command to allow for playback and control of video cels.

 

FKiSS 5 extensions:

FKiSS 5 introduces the mediastart() and mediastop() events to facilitate control of media playback, including movies. Basic FKiSS is deficient in that media playback can be initiated through the sound() or music() action command, but no process exists to recognize when such files stopped or started.
