mouseout() 

Syntax:     mouseout (target)

This event is triggered when the mouse exits a visible image on the screen. The target may be a literal name of an image file or a cel group name or an object group identifier.

The mouseout() event is only triggered once when the mouse leaves the object. It can be triggered again if the mouse enters the object and leaves again.

The correct process is to trigger the event when the mouse no longer overlaps a visible pixel in the image.  Other viewers may trigger this event when the mouse leaves the bounding box area for the image, but one problem with this approach is that the object bounding area may include other cels that are not visible in the object. This can provide a non-intuitive implementation as the mouseout() event may not fire when expected by the user.

Note that visible pixels can include fully transparent pixels that are not the cel transparent color.   

More than one mouseout() event can be specified for a specific file. If sequential event processing is enforced each event will fire in the sequence in which it was defined, otherwise they are queued for concurrent processing. 

 

Related events:  mousein()
Related actions: letmousex(), letmousey()

 

Compatibility:

FKiSS 4   mousout ("picture.cel")  or  mouseout (#3) or mouseout(!Cards) Triggered when the mouse no longer overlaps a visible pixel in an image

 

UltraKiss Options:

MouseInOutBox recognize mouse in/out on visible object boundaries default value "false" (Compatibility option)

 


FKiSS 4 Example: 

;@mouseout("click.cel") ; 
;@ notify("mouse not over cel") ; 

;@mouseout(#3) ; 
;@ notify("mouse not over object group #3") ; 

;@mouseout(!Cards) ; 
;@ notify("mouse not over a card in !Cards") ;
