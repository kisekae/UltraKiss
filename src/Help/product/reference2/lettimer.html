lettimer()

Syntax:     letttimer (variable,alarmname)

The lettimer() statement sets the variable to the amount of time in milliseconds remaining on a scheduled alarm before the alarm is set to fire. If the alarm is not running and is not scheduled then lettimer() returns -1. If the alarm has been activated and is currently running lettimer() returns 0.

Note that there can only be one instance of a specific alarm active at any instance in time. Different alarms can be active at the same time, but the same alarm cannot be active more than once. If an alarm is scheduled with a timer() command and a subsequent timer command is issued for the same alarm, then the new timer delay is applied to the alarm.

Timer delay is specified in milliseconds. Viewers that use a periodic polling method to determine if alarms should be invoked may not be able to determine the exact time at which the alarm will fire. Therefore, time accuracy of lettimer() may not be perfectly precise.

With FKiSS 5 the alarm name can also be specified through a variable that has a string value, providing that the variable name is not the same as an existing alarm name.

 

Related actions: alarm(), randomtimer(), timer(), ifmapped(), ifnotmapped(), iffixed(), ifnotfixed(), ifmoved(), ifnotmoved()

UltraKiss Options:

TimerPeriod the polling period in milliseconds for firing alarms default value "10" (FKiSS option)
EventQueues  the number of concurrent alarms that can run  default value "1" (FKiSS option)
TimerOption enables timer processing default value "true" (FKiSS option)
EventOption  enables alarm processing   default value "true" (FKiSS option)

 

 Example: 

;@lettimer(N,MyAlarm) ; Sets variable N to the time remaining on alarm 'MyAlarm'
