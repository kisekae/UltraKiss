mediastart() 

Syntax:     mediastart (filename)

This event is triggered when a sound file or other media file begins playing. The filename is the name of a media file and must have an extension that is for a recognized file type. The filename should be a literal name enclosed within double quotation marks or a variable that has a string value.

UltraKiss FKiSS 5 introduced the mediastart() and mediastop() events to facilitate control of media playback. Basic FKiSS was deficient in that sound playback could be initiated through the sound() or music() action command, but no process existed to recognize when such files stopped or started.

Note that only one mediastart() and mediastop() event is initiated for media files that repeat.  

More than one mediastart() event can be specified for a specific file. If sequential event processing is enforced each event will fire in the sequence in which it was defined, otherwise they are queued for concurrent processing. 

Related events: mediastop()
Related actions: sound(), music(), mediaplayer(), movie()

 

Compatibility:

FKiSS 5   mediastart ("sound.wav")  or  mediastart ("music.midi") Triggered when the sound begins to play.

 

UltraKiss Options:

CacheAudio retain audio files in memory default value "true" (General option)
CacheVideo retain video files in memory default value "false" (General option)
StopMusic  stops music files on a page change  default value "false" (Media option)

 


FKiSS 5 Example: 

;@EventHandler

;@mediastart("click.wav") ; 
;@ notify("Sound is playing") ; 

;@mediastop("click.wav") ; 
;@ notify("Sound has stopped") ;
