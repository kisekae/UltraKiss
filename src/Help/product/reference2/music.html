music()

Syntax:     music (filename)

The music() command will initiate the playback of an audio stream file. The filename must be a literal specification of a valid path to a sound file or a variable that contains a string path name.

A relative filename is assumed to be relative to the location of the set configuration file. An absolute filename can be specified if desired. The path specifiers for a filename can be '/' or '\'.

KiSS viewers should accept MIDI file encodings. Other encodings are viewer dependent. UltraKiss will accept WAV, AU, RMF, and AIFF encodings for the music() command, as well as MP3 files if appropriately configured.

Multiple music() commands can be concurrently active. A music("") command with an empty filename will terminate all currently playing music files.

If the filename does not exist then the music() command fails silently.

A music() command is independent from a sound() command.

 

FKiSS 5 Syntax:     music (filename[,repeat])

FKiSS 5 introduced an optional repeat parameter for audio playback. This parameter specifies the number of times that the sound is to be repeated. Note that the repeat value specifies the number of sound repetitions, thus a value of 1 repeats the sound once, resulting in the original playback being heard plus one repetition of the sound. A negative repeat value will result in the sound being repeated indefinately.

FKiSS provided no easy way to control or manage sound playback. There was no way to determine when a sound file terminated such that it could be repeated.

The FKiSS5 mediastart() and mediastop() events can be used to recognize sound playback events. These events fire on sound initiation and sound termination for the specified sound file. These events fire on completion of the command and not on completion of the sound. Thus, if repetition was specified, one mediastart event fires at the beginning of the command and one mediastop event fires when all repetitions are complete.

 

Related events: mediastart(), mediastop()
Related actions: sound(), mediaplayer()

 

Compatibility:

FKiSS 1 requires literal filenames. music("happysound.wav")
FKiSS 5 allows string variables let(S,"happysound.wav")
music(S)		  

 

UltraKiss Options:

CacheAudio cache audio files in memory default value "false" (General option)
DebugMedia write audio debug traces to log file default value "false" (Debug option)
StopMusic stops active sound on page change default value "false" (Media option)
VariableCase variable names are case sensitive default value "false" (FKiSS option)

 

FKiSS 1 Example:

;@press(#2) 
;@ music("mysound.mid")                       ; 

 

FKiSS 5 Example:

;@begin() 
;@ music("audio/happymusic.mid",2)      ; play the music three times

;@mediastart("audio/happymusic.mid")    ; event triggered when music starts
;@ notify("happymusic started")  

;@mediastop("audio/happymusic.mid")     ; event triggered when music stops
;@ notify("happymusic stopped")
