exitevent() 

Syntax:     exitevent ()

The exitevent()statement will immediately terminate the execution of the current event module or label module.

If the module had been invoked through a gosub() statement then the exitevent() statement causes a return to the calling module where statement execution continues after the gosub() statement.

If the exitevent() statement is processed inside a label module that was called by a repeat() statement then the label module is exited and the repeat() statement continues with the next repetition of the label module. This differs from an exitloop() statement which both exits the current label module and terminates the repeat() loop.

An exitevent() statement, if processed within a for() or while() loop, immediately exits the module and any remaining loop iterations are not completed.

 

Related actions: gosub(), goto(), event(), repeat(), while(), for()

 

 Example: 

;@ repeat(label1,5,N)

;@ label(label1)
;@ ifequal(N,1) debug("Iteration 1") endif()
;@ ifequal(N,2) debug("Iteration 2 ") endif()
;@ ifequal(N,3) debug("Iteration 3 exitevent()") exitevent() endif()
;@ ifequal(N,4) debug("Iteration 4 ") endif()
;@ ifequal(N,5) debug("Iteration 5 ") endif()
;@ debug("label1 ends")
