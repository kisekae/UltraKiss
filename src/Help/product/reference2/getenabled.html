getEnabled() 

Syntax:     getEnabled(variable,component)

This is a component command to determine if a component is enabled and will respond to events.

The component may be a literal name of any component or a variable that contains the name of a component. Note that the action command name 'getEnabled' is case sensitive and must be specified as shown.

This command sets the return variable to an integer value that represents the component enable state. A value of 1 indicates that the component is enabled and a value of 0 indicates that the component is disabled. Disabled components will not respond to mouse events or accept input. Disabled components may also be displayed in a different color to show their disabled state.

Components are declared as cels within a KiSS configuration as shown in the example below.  Component cel identifiers use specific extensions such as .button, .list, or .textarea to identify the component type.  

#1.9999 List-1.List           : 0     ;%size[146,207] %offset[12,121] %attributes[wrap,vsb]
#1.9999 TextArea-1.TextArea   : 0     ;%size[269,49] %offset[227,176] %attributes[wrap]
#1.9999 Button1.Button        : 0     ;%size[135,25] %offset[325,267] %attributes[text="Press Me"]

The example above defines a List component with vertical scroll bars, a TextArea component that performs word-wrap, and a Button.  The size and location of these components within the object are specified with the %size[] and %offset[] tags.  Component attributes are specified with the %attributes[] tag.  

 

Related actions: setEnabled(), setAttributes(), getAttributes()

 

Compatibility:

UltraKiss Extension getEnabled (B,"button1.togglebutton") Sets the variable B to 1 if the toggle button is enabled

 

Note: The UltraKiss getEnabled() command is viewer specific and not compatible with other viewers.

 

Example: 

#8.999 button1.button         : 0 ;%size[318,26]     %attributes[text="Press Me!"] 
#3.999 list1.list             : 0 ;%size[175,177]    %attributes[vsb]
