setValueAt() 

Syntax:     setValueAt(component,index,value)

This action sets the list component item at the specified index to the given value.

The component may be a literal name of a list or or a variable that contains the name of a list component..

The index may be an literal integer specification or a variable that is an integer or string specification of an integer.

The value may be a literal specification of any type or a variable that is a string specification of some value.

This command fails silently if the specified component is not a list or the index value is out of range. Index values begin at 0.

Components are declared as cels within a KiSS configuration as shown in the example below.  Component cel identifiers use specific extensions such as .button, .list, or .textarea to identify the component type.  

#1.9999 List-1.List           : 0     ;%size[146,207] %offset[12,121] %attributes[wrap,vsb]
#1.9999 TextArea-1.TextArea   : 0     ;%size[269,49] %offset[227,176] %attributes[wrap]
#1.9999 Button1.Button        : 0     ;%size[135,25] %offset[325,267] %attributes[text="Press Me"]

The example above defines a List component with vertical scroll bars, a TextArea component that performs word-wrap, and a Button.  The size and location of these components within the object are specified with the %size[] and %offset[] tags.  Component attributes are specified with the %attributes[] tag.  

 

Related actions: getValueAt(), getIndexOf(), getItemCount()

 

Compatibility:

UltraKiss Extension setValueAt ("title.button","true") sets the specified component selection state

 

Note: The UltraKiss setValueAt() command is viewer specific and not compatible with other viewers.

 

Example: 

#8.999 button1.button         : 0 ;%size[318,26]     %attributes[text="Press Me!"] 
#3.999 list1.list             : 0 ;%size[175,177]    %attributes[vsb]
