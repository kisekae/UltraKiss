setText() 

Syntax:     setText (component,value)

This action sets the component text attribut to the specified string. The component may be a literal name of a button or list or combobox or any other component, or a variable whose value is a string name of a component. The string value may be a literal specification or a variable. Note that the action command name 'setText' is case sensitive and must be speified as shown.

This command fails silently if the specified component does not maintain a text attribute. Component types that maintain text attributes are:
label, textbox, textarea, textfield, passwordfield, button, togglebutton, radiobutton, or any other component to which a text attribute has been assigned,

Components are declared as cels within a KiSS configuration as shown in the example below.  Component cel identifiers use specific extensions such as .button, .list, or .textarea to identify the component type.  

#1.9999 List-1.List           : 0     ;%size[146,207] %offset[12,121] %attributes[wrap,vsb]
#1.9999 TextArea-1.TextArea   : 0     ;%size[269,49] %offset[227,176] %attributes[wrap]
#1.9999 Button1.Button        : 0     ;%size[135,25] %offset[325,267] %attributes[text="Press Me"]

The example above defines a List component with vertical scroll bars, a TextArea component that performs word-wrap, and a Button.  The size and location of these components within the object are specified with the %size[] and %offset[] tags.  Component attributes are specified with the %attributes[] tag.  


Related actions: getText()

 

Compatibility:

UltraKiss Extension setText ("title.button","Press Me!") sets the specified component text

 

Note: The UltraKiss setText() command is viewer specific and not compatible with other viewers.

 

Example: 

#3.999 textfield1.textfield   : 0 ;%size[318,26]     %attributes[text="Item Description"]

;@ setText("textfield1.textfield","To be or not to be") ;
