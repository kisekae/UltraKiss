letkey()

Syntax:     letkey (variable,keyset)

The letkey() statement sets the variable to the character index in the keyset string of the last key pressed, or the value 0 if the key is not in the list.

The keyset parameter can be a string literal or a variable that contains a string. Each character in the string should represent one possible character on the keyboard that can be tested. For example, the string "abcxyz" is a set of six possible characters that are to be tested for keyboard input.

The letkey() startement will also recognize the four cursor control keys 'up', 'down', 'left', and 'right' as special entries within the keyset. For example, the keyset "abcrightleft" recognizes the first three letters of the alphabet plus two cursor control keys, with returned values 1, 2, 3, 4, and 9.

The letkey() statement does not distinguish between upper and lower case characters unless the UltraKiss keyboard case option is set. The letkey() statement is also not capable of recognizing most special keys on the keyboard.

Some viewers may only recognize keyboard entry while a key is actually being pressed. This can lead to viewer compatibility problems as the letkey() statement may only function within keypress() events.

This oddity with cursor keys illustrates the original design problems with FKiSS. The letkey() statement is deprecated in favour of the UltraKiss letkeychar() statement.

 

Related events: keypress(), keyrelease(), keytype()
Related actions: letkeymap(), letkeychar(), letkeycode(), letkeystring(), letkeymodifier()

 

UltraKiss Options:

KeyCase Enable keyboard upper/lower case key entry default value "false" (FKiSS option)
RetainKey Remember last key pressed default value "true" (Compatibility option)

 

 Example: 

;@letkey(N,"a") ; Sets variable N to 1 if the "a" key is pressed, 0 otherwise
;@letkey(N,"012345xyzabcupdown") ; Sets variable N to 10 if the "a" key is pressed
