getItemCount() 

Syntax:     getItemCountf(variable,component)

This is a component command to retrieve the number of items in a list component.

The component may be a literal name of a list component or a combobox component or a variable that contains the name of a list component or combobox component.

Note that the action command name 'getItemCount' is case sensitive and must be specified as shown.

This command returns the integer count of the number of items in the specified list, and -1 if the the component is not a list or combobox. Note that list index values begin at 0.

Components are declared as cels within a KiSS configuration as shown in the example below.  Component cel identifiers use specific extensions such as .button, .list, or .textarea to identify the component type.  

#1.9999 List-1.List           : 0     ;%size[146,207] %offset[12,121] %attributes[wrap,vsb]
#1.9999 TextArea-1.TextArea   : 0     ;%size[269,49] %offset[227,176] %attributes[wrap]
#1.9999 Button1.Button        : 0     ;%size[135,25] %offset[325,267] %attributes[text="Press Me"]

The example above defines a List component with vertical scroll bars, a TextArea component that performs word-wrap, and a Button.  The size and location of these components within the object are specified with the %size[] and %offset[] tags.  Component attributes are specified with the %attributes[] tag.  

 

Related actions: setValueAt(), getValueAt(), getIndexOf()

 

Compatibility:

getIndexOf() 

Syntax:     getIndexOf(variable,component,value)

This is a component command to retrieve the index of a list component that has the specified value. The component may be a literal name of a list component or a combobox component or a variable that contains the name of a list component or combobox component. The value is a literal string or a variable that contains the string representation of the desired item. Note that the action command name 'getIndexOf' is case sensitive and must be specified as shown.

This command returns the integer value of the item in the list that has the specified value, and -1 if the item cannot be found or the component is not a list or combobox. Value comparisons are case sensitive. Note that list index values begin at 0.

Components are declared as cels within a KiSS configuration as shown in the example below.  Component cel identifiers use specific extensions such as .button, .list, or .textarea to identify the component type.

getValueAt() 

Syntax:     getValueAt(variable,component,index)

This is a component command to retrieve the string value of a list component at the specified index. The component may be a literal name of a list or a variable that contains the name of a list component. The index is an integer value or a variable that contains an integer. Note that the action command name 'getValueAt' is case sensitive and must be specified as shown.

This command returns the string valueof the item in the list at the specified index, and an empty string if the specified component is not a list or the index is out of range. Index values begin at 0.

Components are declared as cels within a KiSS configuration as shown in the example below.  Component cel identifiers use specific extensions such as .button, .list, or .textarea to identify the component type.

setValueAt() 

Syntax:     setValueAt(component,index,value)

This action sets the list component item at the specified index to the given value. The component may be a literal name of a list or or a variable that contains the name of a list component.. The index may be an literal integer specification or a variable that is an integer or string specification of an integer. The value may be a literal specification of any type or a variable that is a string specification of some value.

This command fails silently if the specified component is not a list or the index value is out of range. Index values begin at 0.

Components are declared as cels within a KiSS configuration as shown in the example below.  Component cel identifiers use specific extensions such as .button, .list, or .textarea to identify the component type.
