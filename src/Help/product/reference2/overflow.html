overflow() 

Syntax:     overflow ()

This event is triggered on numeric overflow that typically occurs on division by zero. It can be used as an exception handler for various abnormal conditions in your FKiSS code.

The intention of overflow() was to recognize arithmetic error conditions. Most implementations will trap divide by zero fault, but not necessarily arithmetic overflow. This can occur when calculations exceed the capacity of the internal storage location that maintains the result number.

 

Example: 

;@overflow()                ; only fired by OS
;@ notify("Divide by zero") ; show a warning

 

Event execution should stop on overflow faults. The following code should not display the Hello message. The division by zero fault should terminate execution.

;@label(1)                  ;
;@ let(A,1) let(B,0)        ;
;@ div(C,A,B)               ;
;@ notify("Hello!")         ;
