debug() 

Syntax:     debug (literalstring)

The debug() action is intended to display a message dialog on the standard print stream. With FKiSS 1 through FKiSS 4 the string must be a literal value enclosed in double quotation marks. 

 

FKiSS 5 Syntax:     debug (string[,string,string,...])

FKiSS 5 supports string variables, type conversion, and string operations such as concatenation. With FKiSS 5 the argument can be in a variable and multiple arguments can be specified on the debug() statement. Each argument is converted to its string representation and concatenated with all other arguments in the order specified. The resulting string is displayed in a non-modal popup dialog window.

Related actions: notify(), confirm(), showstatus()

 

Compatibility:

FKiSS 1 to 4 only allows literal strings. debug ("This is a message") Prints a debug message on the standard output stream
FKiSS 5 allows variables and concatenation of multiple strings debug("Variable A is: ",A) Prints a debug message on the standard output stream

 

UltraKiss Options:

VariableCase variable names are case sensitive default value "false" (Compatibility option)

 

 

 FKiSS 1 Example:

#1.999 button.cel : 0 ;      
;@press(#1)   debug("Object #1 pressed")  ; 
;@release(#1) debug("Object #1 released")  ; 

 FKiSS 5 Example:

#1.999 button.cel : 0 ;      
;@press(#1)   let(S,"Object #1 pressed") debug(S) ; 
;@release(#1) debug(S," and then released")  ;
