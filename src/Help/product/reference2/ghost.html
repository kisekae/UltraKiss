ghost() 

Syntax:     ghost (target,integer)

This action sets or disables the responsiveness of the target to mouse activities. The target may be a literal name of an image file or a cel group name or an object group identifier. The integer parameteris an on/off switch. A value of 0 clears any ghost behaviour and a non-zero value sets ghost behaviour.

Ghosted images do not respond to mouse activity. Such images cannot be moved with the mouse. Ghosted images will not respond to FKiSS events that require mouse activity such as press(), release(), catch(), drop(), and so on.  Ghosted images can respond to collision events and other events initiated through FKiSS actions.

Ghost operations applied to an object group or a cel group apply to all image cels in the group.


Related actions: altmap(), map(), unmap()

 

Compatibility:

FKiSS 3 allows variables for objects.   ghost(#2,1) ghost (X,0)  Sets or Clears the ghost behaviour of the all cels in the specified object.
FKiSS 4 allows cel group names.   ghost (!Table,1)  Sets ghost behaviour for all cels in the cel group.
FKiSS 5 allows variables for all names let(Cel,"a.cel") ghost (Cel,1)  Sets ghost behaviour of the specified cel, or all cels in the object group or cel group.

 

UltraKiss Options:

AllAmbiguous apply ghost to all ambiguous cels default value "false" (Compatibility option)

 

Example: 

#1.999 button.cel : 0 ;      
#3.999 fk4b3a.cel : 0 ;      normal image
;@press(#1)   ghost(#3,1)             ; the release cannot occur once #1 is ghosted.
;@release(#1) ghost("button.cel",0)   ;
