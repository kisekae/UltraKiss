end()

Syntax: end()

This event is triggered by closing the set. This event fires on a quit() action command or by explicitly closing the set through a menu command.

The end() event is typically used to show termination images and sounds. Most viewers will not process alarms or collision events activated during end() processing .

The end() event is an immediate event and will not be queued for execution. More than one end() event can be specified. If sequential event processing is enforced each event should fire in the sequence in which it was defined. 


Related events: initialize(), begin(), version()
 Related actions: quit()

 

UltraKiss Options:

EventQueues  the number of concurrent events that can run  default value "1" (FKiSS option)
TimerOption enables timer processing default value "true" (FKiSS option)
EventOption  enables alarm processing   default value "true" (FKiSS option)
