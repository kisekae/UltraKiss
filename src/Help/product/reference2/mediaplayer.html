mediaplayer()

Syntax:     mediaplayer (filename[,repeat[,minimized[,x,y]]])

The mediaplayer() command integrates with the UltraKiss Media Player tool used for playback and control of background audio files. The media player is also the preferred method to control display and playback of video files associated with a KiSS set.

For background audio the media player can automatically suspend itself and resume playback if the KiSS set initiates a music() or sound() file. This behaviour is controlled by UltraKiss media options.

Only one mediaplayer() command can be concurrently active. A mediaplayer("") command with an empty filename will terminate all currently playing media files.

The optional repeat parameter is an integer repeat count for playback of the file. A negative repeat count results in continuous looping of the media file.

The optional minimized parameter, if non-zero, hides or minimizes the mediaplayer on invocation. This is the default state. To show the mediaplayer a zero value must be specified for this parameter.

The optional x and y parameters specify the location for the mediaplayer. The default state will center the mediaplayer in the playfield.

Sound files, movie files, or media player playlists can be specified. UltraKiss will accept WAV, AU, RMF, AIFF, MP3, and AVI file encodings for the mediaplayer() command. UltraKiss will also process playback lists which contain entries for many media files. A playlist which is a text file with a ".lst" or ".m3u" extension. A playlist contains one line with the path name of each file to be played, as shown below. Note that in this case the media file locations are relative to the same directory that contains the playlist file.

Era.mp3

Cathar Rhythm.mp3
Mother.mp3
Avemano.mp3

A relative filename is assumed to be relative to the location of the set configuration file. An absolute filename can be specified if desired. The path specifiers for a filename can be '/' or '\'.

If the specified file does not exist then the command fails silently.

A mediaplayer() command is independent from a music() command and a sound() command.

The FKiSS5 mediastart() and mediastop() events can also be used to control playback. These events fire on sound initiation and sound termination for the specified mediaplayer file.

If the mediaplayer file is a playlist then the FKiSS5 mediastart() and mediastop() events fire on sound initiation and sound termination for the playlist.

 

Related events: mediastart(), mediastop()
Related actions: sound(), music()

 

Compatibility:

UltraKiss Extension mediaplayer("HappySong.mid") invokes the mediaplayer tool

 

Note: The UltraKiss mediaplayer() command is viewer specific and not compatible with other viewers.

 

UltraKiss Options:

CacheAudio cache audio files in memory default value "false" (General option)
DebugMedia write audio debug traces to log file default value "false" (Debug option)
StopMusic stops active sound on page change default value "false" (Media option)
VariableCase variable names are case sensitive default value "false" (FKiSS option)

 

FKiSS 5 Example:

;@begin() 
;@ mediaplayer("happymusic.mp3")  ; play the music

;@mediastart("happymusic.mp3")    ; event triggered when music starts
;@ notify("happymusic started")  

;@mediastop("happymusic.mp3")     ; event triggered when music stops
;@ notify("happymusic stopped")
