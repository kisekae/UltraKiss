transparent()

Syntax:     transparent (target,adjustment)

The transparent() statement adjusts the transparency of the target by the adjustment amount. The target can be a cel name, object group number, or cel group name. The adjustment value is a relative transparency integer number ranging from -255 to 255. Positive numbers increase the transparency of the target by the specified amount. Negative numbers decrease the transparency and make the target more opague.

Transparency is a color attribute like red, green, or blue. Color attributes in the ARGB color model are specified as a 4-tuple where each color value ranges from 0 to 255. The transparency or alpha (A) value of the color has valules where 0 is fully transparent and 255 is fully opague. In a similar way, a white color which is maximum red (R), green(G), and blue (B), has an RGB value of 255,255,255. A fully opague white color is 255,255,255,255.

KiSS, however, inverted the alpha interpretation of color transparency. In KiSS, a fully transparent target has a value of 255 and a fully opague target has value 0.

Cels maintain their own individual transparency value. Object and cel groups maintain one transparency value for the group. Relative transparency changes to a group object adjust the object transparency and the relative adjustment is applied to each cel in the object. Other viewers may not maintain group transparency values as a group attribute. This makes it difficult to obtain current group transparencies through a lettransparent() command.

Cel transparency adjustments are applied to all ambiguous cels unless this behaviour is restricted through UltraKiss compatibility options. Otherwise, the first ambiguous cel in the CNF configuration file, possibly restricted to the current page depending on compatibility options, is set.

 

FKiSS 5 Syntax:     transparent (target,adjustment[,bound])

UltraKiss FKiSS5 introduces the ability to adjust transparency outside the practical limits of 0 to 255. Unbounded transparency adjustments allow images to maintain their relative transparency as other image transparencies are applied. This is important for objects and cel groups where individual images each maintain their own transparency. For example, if a group object contains both a shirt and a skirt and the shirt is 50% transparent, then a relative unbounded adjustment of 255 would make the object 100% transparent. The shirt would have a transparency of 150% and the skirt would be 100% transparent. This means that an unbounded reversal of the transparency adjustment would restore the object to its original state where the shirt was 50% transparent and the skirt was fully opague.

The optional 'bound' parameter is an integer number or variable. A value of 0 is used to specify an unbounded transparency adjustment. Bounded adjustments are the default if the parameter is not specified.

FKiSS5 also allows image cel names, object group numbers, or cel group names to be specified in a variable.

 

Related actions: lettransparent()

Compatibility:

FKiSS 1 allows cel and object group targets transparent(#12,100)  
FKiSS 4 allows cel group targets transparent(!Clothing,200)  
FKiSS 5 allows unbounded transparencies and variable targets let(Target,"!Clothing")
transparent(Target,200,0)		  

 

UltraKiss Options:

allambiguous applies transparency adjustments to all ambiguously named cels default value "true" (Compatibility option)
transparentgroup lets object groups and cel groups maintain their own transparency adjustment. The lettransparent() statement returns this value and does not return -1 if the cels in the group have different transparencies. default value "true" (Compatibility option)
contextmap uses the current page context to disambiguate cel choices where ambiguous cels exist. default value "true" (Compatibility option)


Example: 

;@transparent("skirt.cel",100) ; The specified cel becomes more transparent
;@transparent(#12,255) ; The object #12 becomes fully transparent
;@transparent(!Clothing,-255) ; Each cel in the !Clothing group becomes fully opague
;@transparent(Target,-255,0) ; Each cel in the variable group becomes fully opague
