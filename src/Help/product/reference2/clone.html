clone()

Syntax:     clone (variable,object)

The clone() statement creates a clone or duplicate of the specified object. The new object identifier is returned in the variable. The new cloned object contains new references to all the cels in the original object. The cels are all clones of the original group object cels.

The cloned group object is added in the same place to all the pages that the original object used. It is not mapped or visible.

The cloned object and the cloned cels are marked as internal objects to the set configuration. Internal objects are not reported in the CNF file. This means that these object and cels are not permanent and will only exist for the current invocaction.

Further, cloned cels and group objects retain the same draw level as the original object. This means that the cloned object cels are drawn on the screen in the same relatve order as before.

The object must be an object group specifier or a variable containing an object group identifier.

 

Related actions: destroy()

 

 

Compatibility:

FKiSS 5   clone(N,#3)  Variable N is set to the group object number for a new duplicate of object #3.

 

Note: The UltraKiss clone() command is viewer specific and not compatible with other viewers.

 

UltraKiss Options:

VariableCase variable names are case sensitive default value "false" (Compatibility option)

 

 Example: 

;@clone(N,#3) ; N contains the new object number for a clone of object #3
