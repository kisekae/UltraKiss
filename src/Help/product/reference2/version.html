version() 

Syntax:     version (integer)

This event will fire if the viewer supports the FKiSS level specified by the Integer argument. The current FKiSS syntax has five levels: FKiSS, FKiSS2, FKiSS2.1, FKiSS3 and FKiSS4. The proposed FKiSS5 extensions are designed to correct some of the deficiencies in FKiSS coding introduced with FKiSS3 and FKiSS4. 

The level association is as follows:

FKiSS Version Level
FKiSS 1
FKiSS 2 2
FKiSS 2.1 3
FKiSS 3 4
FKiSS 4 5
FKiSS 5 6

The version() event is frequently used to hide warning cels, initially visible if the viewer running the set does not support all of the FKiSS code required.

Clearly, a viewer that supports FKiSS4 should respond to version(1), version(2), version(3), version(4), and version(5) events. However, some viewers are not capable of handling multiple occurrences of  the same event. Thus, due to viewer deficiencies, use of version() can lead to viewer dependent code. 

The version() event is processed after initialize() and before begin(). The version() event is an immediate event and will complete execution before any other event starts. For viewers that can handle multiple events, more than one version() event can be specified for a particular version parameter. If sequential event processing is enforced each event should fire in the sequence in which it was defined. Cels that are mapped and moved within the version() event can fire collision events. 

Related events: initialize(), end(), begin()

 

FKiSS 1 Example:

;@version(3)                  ; only fired if viewer can run FKiSS2.1 or later code
;@ notify("Variables not OK") ; show a warning
;@version(4)                  ; only fired if viewer can run FKiSS3 or later code
;@ notify("Variables OK")     ; show a warning

FKiSS 2 Example:

;@initialize() ; 
;@ unmap(#4) ; 

;@version(1) map("fk1r.cel") ; 
;@version(2) map("fk2r.cel") ; 
;@version(3) map("fk21r.cel") ; 
;@version(4) map("fk3r.cel") ; 
;@version(5) map("fk4r.cel") ; 
;@version(6) map("fk5r.cel") ; 
;@version(7) map("fk6r.cel") ; 

;@begin() ; 
;@ notify("set begins") ;
