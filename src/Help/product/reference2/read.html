read() 

Syntax:     read(status,filename,string,line)

UltraKiss file I/O is designed to support reading and writing of text files in a plain text or encrypted mode. The read() command accesses a previously opened file for reading.

The status parameter must be a variable in which the number of bytes read from the file is returned. For single and double byte characters the byte count is dependent on the file encoding chosen in the UltraKiss options. If there is an error reading the file then the returned status valued will be -1.

The filename parameter is a literal string or a variable that contains the name of the file that has been opened for reading. The filename must be a relative name that refer to files in the same directory or archive file from which the set configuration was loaded. If the file has not been opened then the original status value and returned string remain unchanged.

The string parameter must be a variable. This variable will be set to the value of the text that was read from the specified line of the file.

The line parameter is an interger value specified as a literal value or a variable that identifies the line number in the file to be read. File line numbers begin from 0. If the line does not exit then the returned string will be set to an empty string of length 0, and the status value will be set to -1. Note that the first line in the file begins with line number 0.

When reading a file data is actually read from a text buffer established when the file was first opened. The complete contents of the file were read into the text buffer and all subsequent read() and write() statements access the memory contents of the text buffer.

 

Related actions: open(), write(), close(), edit()

 

Note: The UltraKiss read() command is viewer specific and not compatible with other viewers.

 

 

Compatibility:

UltraKiss Extension read(N,S,12,"myfile.txt") read line 12 of file "myfile.txt"

 

UltraKiss Options:

Encoding  file encoding based upon language    default value "cp1252" (Compatibility option)
