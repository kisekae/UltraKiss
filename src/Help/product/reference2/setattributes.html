setAttributes() 

Syntax:     setAttributes (component,string[,permanent])

This is a component command to set one or more attribute settings for a component, or the value of a specific attribute setting for a component.

The component may be a literal name of any component or video cel.

The string is a literal string or variable specifying a comma delimited list of attribute settings in a 'name=value' format.

The permanent parameter is an optional integer value which if non-zero permanently applies the attribute change to the component. Permanent attributes are maintained across a set restart. By default, attribute settings perfomed through FKiSS commands are not permanent.

Note that the action command name 'setAttributes' is case sensitive and must be specified as shown.

This command will parse and apply attributes specified in a string that is a comma delimited list of all attribute settings in a 'name=value' format.

text The text attribute specifies the initial text that is displayed by components that support text.
read This attribute specifies, for input components, whether the component is read-only. Valid options are True and False. Read-only components cannot respond to user input. Text boxes or textareas in read-only mode cannot accept user input.
wrap This attribute applies to text components. Valid options are True and False. If a text component wraps, words will be wrapped onto the following lines if the text exceeds the width of the component.
align This attribute specifies the initial text alignment. Valid options are Right, Left, and Center. This attribute applies to text boxes or textareas or textfields. Button text is always centered.
noscroll This attribute can be used to disable scrolling for scrollable components. Valid options are True and False. Text boxes, textareas, and lists by default are scrollable. A scrollable component is an input component, whether or not it is read-only. A scrollable component will support both vertical and horizontal scroll bars.
vsb This attribute can be used to force the display of a vertical scroll bar for a scrollable component. Valid options are True and False. 
hsb This attribute can be used to force the display of a horizontal scroll bar for a scrollable component. Valid options are True and False. 
multiple This attribute enables multiple selections for list components. Valid options are True and False. 
disabled This attribute sets the initial enable state for a component. If a component is disabled it cannot accept user input action. Valid options are True and False.
selected This attribute sets the initial selection state for a component, such as a checkbox or radio button. Valid options are True and False. 
nomargin This attribute applies to button component types. Valid options are True and False. If a no margin attribute is set text can extend to the boundaries of the button.
fontname This attribute lets you select the text font from the fonts available on your system. 
fontsize This area lets you specify a non-standard font size, in points.  Typical values are 8, 10, 11, 12, 14, 16, 18, and so on.
fontstyle This attribute specifies the style of type, either bold or italic or plain.
border This attribute specifies the style of border to be drawn around the component. This is either line, bevel, etched, matte, or titled. Borders without type default to line borders. Depending on the border type, the border attributes may differ. Line borders have a thickness attribute. Titled borders have a title string. Beveled borders have a bevel type.
thickness This attribute applies to line type borders. It specifies the thickness of the line in pixels.
bevelstyle This attribute applies to bevel type borders. Beveled borders may be raised or lowered.
bordertitle This attribute applies to titled borders. This is the title text for the border.
insets This attribute specifies the border spacing around the top, left, bottom, and right sides of the component. The spacing is in pixels.
bdc This attribute specifies the primary color for the border. The value is an integer that specifies RGB colors.
bc This attribute specifies the background color for the component. The value is an integer that specifies RGB colors.
fc This attribute specifies the foreground color for the component. The value is an integer that specifies RGB colors.

 

Components are declared as cels within a KiSS configuration as shown in the example below.  Component cel identifiers use specific extensions such as .button, .list, or .textarea to identify the component type.  

#1.9999 List-1.List           : 0     ;%size[146,207] %offset[12,121] %attributes[wrap,vsb]
#1.9999 TextArea-1.TextArea   : 0     ;%size[269,49] %offset[227,176] %attributes[wrap]
#1.9999 Button1.Button        : 0     ;%size[135,25] %offset[325,267] %attributes[text="Press Me"]

The example above defines a List component with vertical scroll bars, a TextArea component that performs word-wrap, and a Button.  The size and location of these components within the object are specified with the %size[] and %offset[] tags.  Component attributes are specified with the %attributes[] tag.

Related actions: getAttributes(), getEnabled(), setEnabled()

 

Compatibility:

UltraKiss Extension setAttributes ("button1.togglebutton","bc=27345") Sets the background color of the toggle button

 

Note: The UltraKiss setAttributes() command is viewer specific and not compatible with other viewers.

 

Example: 

#3.999 button1.button         : 0 ;%size[318,26]     %attributes[fc=0,text="Press Me!"] 

;@press("button1.button") ; 
;@ getAttributes(S,"button1.button","text") ; 
;@ indexof(N,S,"No")                        ; test text value
;@ iflessthan(N,0)                          ;
;@ let(S,"fc=255,text='No, Press Me!'")     ;
;@ else()                                   ;
;@ let(S,"fc=0,text='Press Me!'")           ;
;@ endif()                                  ;
;@ setAttributes("button1.button",S,1)      ; permanent change
