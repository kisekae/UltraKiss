UltraKiss Components

UltraKiss V1.0 (beta 5) introduced new capability to include basic Graphical User Interface (GUI) components within a KiSS set.  Components can be used for interactive user input and display of information and provide an easy way to create standard objects on the screen.

Components are either active input components or passive components.  Active input components, such as a multi-line Text Area or a single line Text Field are capable of accepting user input or selection.  Passive components, such as a standard Button or a Text Area that has the read attribute assigned do not accept input and are incapable of responding to user events.  Passive components behave in a similar way to a KiSS image cel.  These components can be drawn, layered, and positioned on the screen along with other KiSS images.  Conversely, active input components intercept mouse events and cannot be directly manipulated with mouse movements.  Input components are drawn on top of all other images in the KiSS set and can extend beyond the playfield boundaries.

Components are declared as cels within a KiSS configuration as shown in the example below.  Component cel identifiers use specific extensions such as .button, .list, or .textarea to identify the component type.  

#3.9999 Add.gif            : 18     ;%offset[167,176]
#3.9999 Next.gif           : 18     ;%offset[399,377]
#3.9999 Back.gif           : 18     ;%offset[339,377]
#3.9999 Cancel.gif         : 18     ;%offset[273,378]
#3.9999 LI0S8-1.List       : 18     ;%size[146,207] %offset[12,121] %attributes[wrap,vsb]
#3.9999 TA0S9-1.TextArea   : 18     ;%size[269,49] %offset[227,176] %attributes[wrap]
#3.9999 TA0Obj.TextArea    : 18     ;%size[321,65] %offset[217,11] %attributes[read,wrap]
#3.9999 Action2Date.Button : 18     ;%size[135,25] %offset[325,267]
#3.9999 Objective10.gif    : 18     ;

In this example object #3 contains a List, two TextAreas, and a Button.  The size and location of these component within the object are specified with the %size[] and %offset[] tags.  Component attributes are specified with the %attributes[] tag.  

 

Component Attributes

Every component has a set of attributes that specify the presentation characteristics for the object. Component attributes are specified with a %attributes[...] tag. To facilitate the specification of component attributes a dialog as shown in Figure 1 below can be used.  This dialog can be accessed through the object context menu, selected by right-clicking while over a componentobject when UltraKiss is in Edit mode.

 

Figure 1

 

Text:  The text attribute specifies the initial text that is displayed by components that support text.

Read: This attribute specifies, for input components, whether the component is read-only. Valid options are True and False. Read-only components cannot respond to user input. Text boxes or textareas in read-only mode cannot accept user input.

Wrap: This attribute applies to text components. Valid options are True and False. If a text component wraps, words will be wrapped onto the following lines if the text exceeds the width of the component.

Align: This attribute specifies the initial text alignment. Valid options are Right, Left, and Center. This attribute applies to text boxes or textareas or textfields. Button text is always centered.

No scroll:  This attribute can be used to disable scrolling for scrollable components. Valid options are True and False. Text boxes, textareas, and lists by default are scrollable. A scrollable component is an input component, whether or not it is read-only. A scrollable component will support both vertical and horizontal scroll bars.

Vertical scroll: This attribute can be used to force the display of a vertical scroll bar for a scrollable component. Valid options are True and False. 

Horizontal scroll: This attribute can be used to force the display of a horizontal scroll bar for a scrollable component. Valid options are True and False. 

Multiple:  This attribute enables multiple selections for list components. Valid options are True and False. 

Disabled:  This attribute sets the initial enable state for a component. If a component is disabled it cannot accept user input action. Valid options are True and False. 

Selected:  This attribute sets the initial selection state for a component, such as a checkbox or radio button. Valid options are True and False. 

No margin:  This attribute applies to button component types. Valid options are True and False. If a no margin attribute is set text can extend to the boundaries of the button.

Font name:  This attribute lets you select the text font from the fonts available on your system. 

Font size:  This area lets you specify a non-standard font size, in points.  Typical values are 8, 10, 11, 12, 14, 16, 18, and so on.

Font style:  This attribute specifies the style of type, either bold or italic or plain.

Border: This attribute specifies the style of border to be drawn around the component. This is either line, bevel, etched, matte, or titled. Borders without type default to line borders. Depending on the border type, the border attributes may differ. Line borders have a thickness attribute. Titled borders have a title string. Beveled borders have a bevel type.

Thickness: This attribute applies to line type borders. It specifies the thickness of the line in pixels.

Bevel type: This attribute applies to bevel type borders. Beveled borders may be raised or lowered.

Title: This attribute applies to titled borders. This is the title text for the border.

Insets:  This attribute specifies the border spacing around the top, left, bottom, and right sides of the component. The spacing is in pixels.

Border color:  This attribute specifies the primary color for the border. The Reset button may be used to remove any previously specified border color.

Back color:  This attribute specifies the background color for the component. The Reset button may be used to remove any previously specified background color.

Fore color:  This attribute specifies the foreground color for the component. The Reset button may be used to remove any previously specified foreground color.

 

 

Component Events

FKiSS press() and release() events can be attached to all component objects.  These events fire when the user interacts with a component to either press a button, change the state of a checkbox, or select an item from a list or combo box.  For example, the following events are fired if the user presses the button component show above or selects an item from the list component.

;@press("Action2Date.Button") 
;@ notify("Button pressed")  

;@release("LI0S8-1.List") 
;@ getSelectedValue(S,"LI0S8-1.List")
;@ notify("User selected ",S) 

 

Component Conversion to Cel Images

For compatibility, any read-only component such as a button, label, or textbox can be converted to a cel image. Input components such as lists and user input text areas cannot be represented as cels as these components require active user input.  To convert label components and buttons to cel images, set the Options-Export-Export GUI Components as Cel Files option.
