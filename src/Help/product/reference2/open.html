open() 

Syntax:     open(status,filename,mode[,decode])

UltraKiss file I/O is designed to support reading and writing of text files in a plain text or encrypted mode. The open() command is used to open a named text file for reading or writing.

The status parameter must be a variable in which the number of bytes in the file is returned. For single and double byte characters the byte count is dependent on the file encoding chosen in the UltraKiss options. If there is an error opening the file then the status returned is -1.

The filename parameter is a literal string or a variable that contains the name of the file to be opened. The filename must be a relative name that refer to files in the same directory or archive file from which the set configuration was loaded.

The mode parameter is a string literal or variable that determies whether the file is opened for reading or writing, or both. The mode parameter must be specified as "w" for writing, "r" for reading, or "rw" for reading and writing. If a file is opened for reading then the file will be referenced from the set source directory or archive file depending on how the set was loaded. For example, if the KiSS set is packaged in an archive file then the file to be opened must also exist in the archive. If a file is opened for writing then the file will always be written to the local directory from which the configuration was loaded. If the file is opened for read-write mode then the file is first referenced from the local directory and if this file does not exist then the file will be referenced from the set archive file and written to the local directory when closed.

The optional decode parameter is used to specify that MD5 decription is required for the file contents. If the decode parameter is non-zero then the file will be decrypted on open such that the contents can be read.

Files open in write mode will erase the current contents of any existing file. A file opened in read-write mode will retain the contents of any existing file.

A file open command reads the complete file contents into a text buffer. Subsequent reads and writes to the file update the text buffer contents. The buffer may be written to its final destination when the file is closed.

 

Related actions: read(), write(), close(), edit()

 

Note: The UltraKiss open() command is viewer specific and not compatible with other viewers.

 

 

Compatibility:

UltraKiss Extension open(N,"myfile.txt","r") open file "myfile.txt" for reading

 

UltraKiss Options:

Encoding  file encoding based upon language    default value "cp1252" (Compatibility option)

 

 Example: 

;@press(#1) ;
;@ open(N,"test.txt","w") ;
;@ notify("File test.txt opened for write, status ",N) ;

;@press(#4) ;
;@ open(N,"test.txt","r") ;
;@ notify("File test.txt opened for read, status ",N) ;
