changeset() 

Syntax:     changeset (integer)

This action programattically changes the current page of the KiSS set. Pages are numbered beginning from 0. Note that this command will cause all defined set() events to fire for the specified page, and page set changes can cause color set changes for associated col() events.

The changeset() command should not be used in initialize() events, but can be used in begin() events.

Any attempt to change to a page that exceeds the maximum number of pages available in the KiSS set will fail silently.

 

Related actions: set(), changecol(), letmaxpage()

 

Compatibility:

FKiSS 1 allows cel literal names or objects. changeset (3) Sets the active page to the fourth page in the set.
FKiSS 3 allows variables for objects.   let(X,10) changeset (X)  Sets the active page to the value in variable X.
FKiSS 5 allows an unlimited number changeset(25) Previous FKiSS versions were limited to a maximum 10 pages.

 

 Example: 

%palette0.kcf

#1.999 button.cel : 0 ;      
#3.999 fk4b3a.cel : 0 ;     

;@changeset(3)    ; 


Run this example.
