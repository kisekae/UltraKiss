letinside()

Syntax:     letinside (variable,target1,target2)

The letinside() statement returns the value of 1 in the specified variable if the bounding boxes of the two target entities overlap. If the entities do not overlap a zero value is returned. A bounding box is the rectangle that encloses the image.

The target entities can be a cel name, an object group specifier, or a cel group specifier. For object groups or cel groups the entities overlap if the bounding box from any cel in the first target overlaps the bounding box from any cel in the second target. Target parameters can be specified as literal values or as variables.

Cels that are ghosted can overlap with other entities. Cels that are visible yet fully transparent can overlap with other entities.

Where greater accuracy is required use the letcollide() statement to identify overlaps between individual visible pixels.


Related events: in(), out()

Related actions: letcollide()

 

Compatibility:

FKiSS 4   letinside(N,"shirt.cel",#1)  N becomes 1 if "shirt.cel" overlaps any visible cel in object #1

 

UltraKiss Options:

AllAmbiguous apply test to all ambiguous cels default value "false" (Compatibility option)
ContextMap use page context to disambiguate cels default value "true" (Compatibility option)
VariableCase variable names are case sensitive default value "false" (Compatibility option)

 

 Example: 

;@letinside (N,"shirt.cel",#1) ; N is 1 if overlaps any visible cel in object #1
;@letinside (N,!Clothes,!Doll) ; N is 1 if any clothing cel overlaps any doll cel
