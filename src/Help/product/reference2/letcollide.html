letcollide()

Syntax:     letcollide (variable,target1,target2)

The letcollide() statement returns the value of 1 in the specified variable if the two target entities overlap at any pixel. If the entities do not overlap a zero value is returned.

The target entities can be a cel name, an object group specifier, or a cel group specifier. For object groups or cel groups the entities overlap if any pixel from any visible cel in the groups overlap. Target parameters can be specified as literal values or as variables.

Pixels that match the cel transparent color do not overlap. Pixels that are not the cel transparent color yet are fully transparent do overlap. Thus, a cel made fully transparent with a transparent() command can overlap with another cel. Cels that are ghosted can also overlap with other entities.

Pixel collision determination is computationally intensive. Where pixel accuracy is not required use the letinside() statement to identify overlaps between bounding boxes.


Related events: collide(), apart()

Related actions: letinside()

 

Compatibility:

FKiSS 4   letcollide (N,"shirt.cel",#1)  N becomes 1 if "shirt.cel" overlaps any visible cel in object #1

 

UltraKiss Options:

AllAmbiguous apply test to all ambiguous cels default value "false" (Compatibility option)
ContextMap use page context to disambiguate cels default value "true" (Compatibility option)
VariableCase variable names are case sensitive default value "false" (Compatibility option)

 

 Example: 

;@letcollide (N,"shirt.cel",#1) ; N is 1 if overlaps any visible cel in object #1
;@letcollide (N,!Clothes,!Doll) ; N is 1 if any clothing cel overlaps any doll cel
