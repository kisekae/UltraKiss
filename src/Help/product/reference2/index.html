Introduction

KiSS (Kisekae Set System) is an image processing system used to create computer representations of two-dimensional virtual dolls. FKiSS is an event language that facilitates the programming of automated actions and system interaction with the user. This document describes the KiSS configuration file format and the syntax and semantics of the FKiSS programming language.

UltraKiss extended and standardized many aspects of the FKiSS language. The standardization, refered to as FKiSS version 5, was designed to correct many of the FKiSS language deficiencies. Each UltraKiss FKiSS command is either a backward compatible extension to previous versions of FKiSS (FKiSS5), or is a new command implemented for completeness or new capabilities (FKiSS5 Extension), or is an UltraKiss viewer specific command (UltraKiss Extension).

 

The FKiSS Language

Data Types

The FKiSS programming language allows programmed capabilities when developing KiSS sets. The language was originally conceived as a non-procedural language that would respond to various events, but with the development of FKiSS 3 a procedural structure was imposed that introduced program variables and control logic.


With FKiSS 5, the FKiSS programming language provides integer and real number arithmetic, string and character manipulation, nested if-statements for logic control, subroutines and function capable label blocks, with local and global variables to manage state information.

The KiSS Configuration (CNF) File

Titles and Comments

The KiSS set configuration file is the control file that identifies the images and objects that compose a KiSS set. This file must exist in every KiSS set.


UltraKiss automatically maintains and generates the CNF file to eliminate much of the manual efforts previously required by KiSS artists.


Edit changes to the CNF file are automatically applied to the current KiSS set. This edit-test process is an improved development cycle from earlier KiSS viewers.
