write() 

Syntax:     write(status,filename,string)

UltraKiss file I/O is designed to support reading and writing of text files in a plain text or encrypted mode. The write() command appends a line of text to a previously opened file.

The status parameter must be a variable in which the number of bytes written to the file is returned. For single and double byte characters the byte count is dependent on the file encoding chosen in the UltraKiss options. If there is an error writing the file then the returned status valued will be -1.

The filename parameter is a literal string or a variable that contains the name of the file that has been opened for writing. The filename must be a relative name that refer to files in the same directory or archive file from which the set configuration was loaded. If the file has not been opened then the original status value and returned string remain unchanged.

The string parameter must be a literal string constant or a variable that contains a string value.

When writing a file the data is actually written to a text buffer that was established when the file was first opened. The actual writing of the text to permanent storage does not occur until the file is closed with a commit option specified.

A write() command appends text to the end of the file. A read() command returns lines of text in the file. Therefore, write() commands should include newline characters ('\n') in the data to properly terminate each line of text. Failure to do this will result in a single line of text in the file.

 

Related actions: open(), read(), close(), edit()

 

Note: The UltraKiss write() command is viewer specific and not compatible with other viewers.

 

 

Compatibility:

UltraKiss Extension write(N,"this is text","myfile.txt") write the text to file "myfile.txt"

 

UltraKiss Options:

Encoding  file encoding based upon language    default value "cp1252" (Compatibility option)
