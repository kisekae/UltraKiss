randomtimer()

Syntax:     randomtimer (alarmname,delay,range)

The randomtimer() statement schedules an alarm of the specified name to begin execution after a random delay, in milliseconds, is computed using the formula (delay + rnd()*range), where the rnd() function returns a random fraction between 0 and 1. The alarm delay, if 0, can be used to cancel or stop any future alarm execution that has been previously scheduled.

Note that there can only be one instance of a specific alarm active at any instance in time. Different alarms can be active at the same time, but the same alarm cannot be active more than once. If an alarm is scheduled with a randomtimer() command and a subsequent timer command is issued for the same alarm, then the new timer delay is applied to the alarm.

Timer delay is specified in milliseconds. Viewers that use a periodic polling method to determine if alarms should be invoked may not fire the alarm at the exact millisecond specified. These viewers should process the alarm on their next poll cycle. Therefore, time accuracy of alarm firing may not be perfectly precise.

Note that there is no defined order for event processing in KiSS, thus the sequence of alarms that are supposed to start at the same time is indeterminate. Single threaded viewers that do not support concurrent events may process alarm() events in the order in which timer() commands were processed, however this assumption can make your code viewer dependent.

 

FKiSS 5 Syntax:     randomtimer (alarmname,delay,range[,operand1,operand2,operand3,...])

UltraKiss FKiSS5 introduces the ability to pass parameters to the alarm code module, similar to parameters on label modules. The optional parameter list specifies a set of variables or literal constants that can be referenced as local variables within the alarm code module. With FKiSS 5 the alarm name can also be specified through a variable that has a string value, providing that the variable name is not the same as an existing alarm name.

 

Related actions: alarm(), timer(), lettimer(), ifmapped(), ifnotmapped(), iffixed(), ifnotfixed(), ifmoved(), ifnotmoved()

Compatibility:

FKiSS 1 allows numeric alarm identifiers. randomtimer(12,100,500) alarm 12 triggered in 100 to 600 ms
FKiSS 2 allows named identifiers.   randomtimer(Ring,200,0) alarm 'Ring' triggered in 200 ms
FKiSS 5 allows alarm parameters randomtimer(Ring,200,500,X,Y) alarm 'Ring' triggered in 200 to 700 ms with arguments X and Y passed to the alarm.

 

UltraKiss Options:

TimerPeriod the polling period in milliseconds for firing alarms default value "10" (FKiSS option)
EventQueues  the number of concurrent alarms that can run  default value "1" (FKiSS option)
TimerOption enables timer processing default value "true" (FKiSS option)
EventOption  enables alarm processing   default value "true" (FKiSS option)

 

 Example: 

;@randomtimer(12,1000,5000) ; Starts alarm 12 at a random time between 1 and 6 seconds  
 ;@randomtimer(Alarm1,N,5000) ; Starts 'Alarm1' at a random time between N and N+5 seconds  
;@randomtimer(FYS,1000,0,Param1,"P2") ; Starts 'FYS' in 1 second passing parameters
