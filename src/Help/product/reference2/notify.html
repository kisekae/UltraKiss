notify() 

Syntax:     notify (literalstring)

The notify() action is intended to display a message dialog on the screen. With FKiSS 1 through FKiSS 4 the string must be a literal value enclosed in double quotation marks. 

 

FKiSS 5 Syntax:     notify (string[,string,string,...] )

FKiSS 5 supports string variables, type conversion, and string operations such as concatenation. With FKiSS 5 the argument can be in a variable and multiple arguments can be specified on the notify() statement. Each argument is converted to its string representation and concatenated with all other arguments in the order specified. The resulting string is displayed in a non-modal popup dialog window.

Notify() dialogs do not suspend dynamic execution. Multiple notify windows can be concurrently displayed. For dialogs that must wait for a response, use the confirm() statement.


Related actions: debug(), confirm(), showstatus()

 

Compatibility:

FKiSS 1 to 4 only allows literal strings. notify ("This is a message") Displays a debug message in a dialog window.
FKiSS 5 allows variables and concatenation of multiple strings notify("Variable A is: ",A) Displays a composite message in a dialog window.

 

UltraKiss Options:

VariableCase variable names are case sensitive default value "false" (Compatibility option)

 

 

 FKiSS 1 Example:

#1.999 button.cel : 0 ;      
;@press(#1)   notify("Object #1 pressed")  ; 
;@release(#1) notify("Object #1 released")  ; 

 FKiSS 5 Example:

#1.999 button.cel : 0 ;      
;@press(#1)   let(S,"Object #1 pressed") notify(S) ; 
;@release(#1) notify(S," and then released")  ;
