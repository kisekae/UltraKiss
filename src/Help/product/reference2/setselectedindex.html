setSelectedIndex() 

Syntax:     setSelectedIndex(component,index)

This action sets the selected item in the list component or combobox component to the specified index value. The component may be a literal name of a list or or a variable that contains the name of a list component or combobox component. The index may be an literal integer specification or a variable that is an integer.

For list or combobox components that have the 'multiple' attribute set to allow multiple selections, the setSelectedIndex() command will allow for multiple selections if applied to different indicies in the component.

If the specified index is out of range for the component then all currently selected items are cleared.

This command fails silently if the specified componentis not a list or combobox. Index values begin at 0.

Components are declared as cels within a KiSS configuration as shown in the example below.  Component cel identifiers use specific extensions such as .button, .list, or .textarea to identify the component type.  

#1.9999 List-1.List           : 0     ;%size[146,207] %offset[12,121] %attributes[wrap,vsb]
#1.9999 TextArea-1.TextArea   : 0     ;%size[269,49] %offset[227,176] %attributes[wrap]
#1.9999 Button1.Button        : 0     ;%size[135,25] %offset[325,267] %attributes[text="Press Me"]

The example above defines a List component with vertical scroll bars, a TextArea component that performs word-wrap, and a Button.  The size and location of these components within the object are specified with the %size[] and %offset[] tags.  Component attributes are specified with the %attributes[] tag.  

 

Related actions: getSelectedIndex(), getNextSelectedIndex(), getSelected(), setSelected(), getSelectedValue(), setSelectedValue(), getSelectedItem(), setSelectedItem()

 

Compatibility:

UltraKiss Extension setSelectedIndex ("list1.list",-1) unselects all selections in list1.list

 

Note: The UltraKiss setSelectedIndex() command is viewer specific and not compatible with other viewers.

 

Example: 

#8.999 button1.button         : 0 ;%size[318,26]     %attributes[text="Press Me!"] 
#3.999 list1.list             : 0 ;%size[175,177]    %attributes[vsb] 

;@initialize() ;
;@ addItem("list1.list","carrots") ; index 0
;@ addItem("list1.list","potatoes") ; index 1
;@ addItem("list1.list","corn") ; index 2
;@ addItem("list1.list","peas") ; index 3
;@ addItem("list1.list","apple, orange, grape, cherry, plum, pear, peach") ;

;@press(#8) ;
;@ setSelectedIndex("list1.list",Index) ; unselects everything if Index > 4 or Index < 0
;@ add(Index,Index,1) ;
;@ getSelectedIndex(S,"list1.list") ;
;@ concat(S,"List selected ",S) ;
;@ notify(S) ;

 

getSelectedValue() 

Syntax:     getSelectedValue(variable,component)

This is a component command to retrieve the value of a list component or combobox component that has a selected item. The component may be a literal name of a list or combobox, or a variable that contains the name of a list component or combobox component. The value of a selected item is the string representation of the item in the list. Note that the action command name 'getSelectedValue' is case sensitive and must be specified as shown.

This command sets the specified variable to the string value of the first selected item index in the list or combobox. If there is no currently selected item in the list or if the component is not a list or combobox the variable is set to an empty string. Note that indexing of list items begins at 0.

Components are declared as cels within a KiSS configuration as shown in the example below.  Component cel identifiers use specific extensions such as .button, .list, or .textarea to identify the component type.

#1.9999 Button1.Button        : 0     ;%size[135,25] %offset[325,267] %attributes[text="Press Me"]

The example above defines a List component with vertical scroll bars, a TextArea component that performs word-wrap, and a Button.  The size and location of these components within the object are specified with the %size[] and %offset[] tags.  Component attributes are specified with the %attributes[] tag.

setSelectedValue() 

Syntax:     setSelectedValue(component,value)

This action selects the item in the list component or combobox component providing that the string value is a member of the list. The component may be a literal name of a list or or a variable that contains the name of a list component. The value may be a string literal specification or a variable that has a string value.

The comparision for selection is case sensitve. The item must be in the list exactly as specified for it to be selected. For list components that have the 'multiple' attribute set to allow multiple selections, the setSelectedValue() command will allow for multiple selections if applied to different items in the component.

This command fails silently if the specified component is not a list component or a combobox component.

Components are declared as cels within a KiSS configuration as shown in the example below.  Component cel identifiers use specific extensions such as .button, .list, or .textarea to identify the component type.

The example above defines a List component with vertical scroll bars, a TextArea component that performs word-wrap, and a Button.  The size and location of these components within the object are specified with the %size[] and %offset[] tags.  Component attributes are specified with the %attributes[] tag.  

 

Related actions: getSelectedValue(), getSelectedIndex(), getNextSelectedIndex(), setSelectedIndex(), getSelected(), setSelected(), getSelectedItem(), setSelectedItem()

 

Compatibility:

setSelectedItem() 

Syntax:     setSelectedItem(component,value)

This action selects the item in the combobox component that has the specified string value. The component may be a literal name of a combobox or or a variable that contains the name of a combobox component. The value may be a string literal specification or a variable that has a string value. For selection of items by value in a combobox use setSelectedItem().

The comparision for selection is case sensitve. The item must be in the combobox exactly as specified for it to be selected. For combobox components that have the 'multiple' attribute set to allow multiple selections, the setSelectedItem() command will allow for multiple selections if applied to different items in the component.

This command fails silently if the specified component is not a combobox.

Components are declared as cels within a KiSS configuration as shown in the example below.  Component cel identifiers use specific extensions such as .button, .list, or .textarea to identify the component type.

The example above defines a List component with vertical scroll bars, a TextArea component that performs word-wrap, and a Button.  The size and location of these components within the object are specified with the %size[] and %offset[] tags.  Component attributes are specified with the %attributes[] tag.
