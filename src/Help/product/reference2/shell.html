shell() 

Syntax:     shell (string)

The shell() action will initiate the execution of an external program on the host system. There is no indication if the shell command is successful.

If the string is a valid URL with protocols "http:" or "file:" or "mailto:" the default system browser is launched for the specified URL. Note that browser commands are not constrained by the EnableShell option that permits or restricts external shell program execution.

 

FKiSS 5 Syntax:     shell (string[, exitcode])

FKiSS 5 syntax provides for an optional integer exitcode variable. If the process is not launched the value returned is -1, otherwise the FKiSS event waits until the external process terminates.  


Compatibility:

FKiSS 1 to 4 only allows literal strings. shell ("notepad c:/myfile.txt") Invoke an external program
FKiSS 5 allows variables and an optional return state let(A,"ls -l ") shell(A,ExitCode) Invoke an external program and wait

 

UltraKiss Options:

EnableShell allow shell command to execute default value "false" (Compatibility option)

 

 

 FKiSS 1 Example:

;@press(#1)   shell("mailto:somebody@mydomain.com")  ; 

 FKiSS 5 Example:

;@press(#1)   let(S,"echo Hello World!") shell(S) ; 
;@release(#1) shell("http://www.kisekaeworld.com")  ;
