setSelected() 

Syntax:     setSelected(component,boolean)

This action sets the component selected state to the specified state. The component may be a literal name of a button or togglebutton or menu item. The boolean may be an literal integer specification, a literal string specification, or a variable that is an integer or string. An integer value of 0 clears the selection state and a non-zero value will set the selection state. A string value of "1" or "true" will set the selection state and any other string value will clear the selection state. Note that the action command name 'setSelected' is case sensitive and must be specified as shown.

This command fails silently if the specified component does not maintain a selection attribute.

Components are declared as cels within a KiSS configuration as shown in the example below.  Component cel identifiers use specific extensions such as .button, .list, or .textarea to identify the component type.  

#1.9999 List-1.List           : 0     ;%size[146,207] %offset[12,121] %attributes[wrap,vsb]
#1.9999 TextArea-1.TextArea   : 0     ;%size[269,49] %offset[227,176] %attributes[wrap]
#1.9999 Button1.Button        : 0     ;%size[135,25] %offset[325,267] %attributes[text="Press Me"]

The example above defines a List component with vertical scroll bars, a TextArea component that performs word-wrap, and a Button.  The size and location of these components within the object are specified with the %size[] and %offset[] tags.  Component attributes are specified with the %attributes[] tag.  


Related actions: getSelected(), getSelectedIndex(), getNextSelectedIndex(), setSelectedIndex(), getSelectedValue(), setSelectedValue(), getSelectedItem(), setSelectedItem()

 

Compatibility:

UltraKiss Extension setSelected ("title.button","true") sets the specified component selection state

 

Note: The UltraKiss setSelected() command is viewer specific and not compatible with other viewers.

 

Example: 

#3.999 b1.togglebutton   : 0 ;%size[318,26]     %attributes[text="Item Description"] 

;@press("button1.cel") ; 
;@ setSelected("b1.togglebutton",1) ;
