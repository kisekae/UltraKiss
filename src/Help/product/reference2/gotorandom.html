gotorandom() 

Syntax:     gotorandom (percent,labelname1,labelname2)

The gotorandom statement will transfer control to either the labelname1 module or the labelname2 module randomly, such that the specified average percent of random choices will be to labelname1. There is no return to the code module with the gotorandom() statement. Any following ststements will not be executed.

The percent parameter should be an integer literal value or a variable with an integer value between 0 and 100. If the percent value is less than or equal to 0 then all random choices will invoke labelname2. If the percent value is greater than or equal to 100 then all random choices will be to labelname1.

 

FKiSS 5 Syntax:     gotorandom (percent,labelname1,labelname2[, argument1, argument2, ...])

For the gotorandom() command it is recommended that the labelname1 and labelname2 modules use a consistent and equivalent parameter set. The set of arguments can be applied to either the labelname1 module or the labelname2 module depending on which is randomly chosen. If the parameter set is not consistent then incorrect or missing parameter values will be established for one or the other label modules.

The optional argument list specifies a set of literal constants or variables that can be referenced as parameters within the label module.

Local variable names are prefixed with an '@' sign. Thus, the variable '@N' is a local variable declared and referenced within the module and is distinct from any global variable 'N'. Calling the label module recursively creates new instances of all local variables distinct from any other local variable of the same name.

Return values can be specified for label modules by assigning a value to a local variable of the same name as the label module.

 

Related actions: gosub(), goto(), gosubrandom(), label

VariableCase variable names are case sensitive default value "false" (Compatibility option)
