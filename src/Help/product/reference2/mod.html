mod()

Syntax:     mod (result,operand1,operand2)

This command performs an arithmetic division. The result variable is is assigned the remainder of operand1 divided by operand2.  The result must be a variable. Operand1 and operand2 can be numeric literal values or variables that represent numeric values.

result = operand1 mod operand2. 

The mod function is defined such that the result will be negative only if operand1 is negative, and positive otherwise.  The magnitude of the result is always less than operand2. The mod function on floating-point operators will behave in a manner analogous to that of the integer remainder operator; this may be compared with the C library function fmod.

FKiSS 3 and FKiSS 4 only support Integer values that range from -2147483648 to 2147483647, inclusive.  UltraKiss FKiSS 5 provides support for floating point numbers and long integers that can range from -9223372036854775808 to 9223372036854775807, inclusive. With FKiSS5 implicit conversion between variables is automatically performed and all calculations maintain the appropriate precision. 

Integer division is performed if both operand1 and operand2 are of integer or long type.  Floating point division is performed if either operand1 or operand2 is a fractional number. To force floating point division when both operands are integer type, convert one operand to floating point by first adding the constant value 0.0.  To force integer division when either operand is floating point, first perform a floating point division and then convert the quotient to an integer with the math() function.

Numeric overflow may occur if the resulting difference cannot be accurately represented by the computer for integer division. Evaluation of a floating-point remainder operator never throws a run-time exception, even if the right-hand operand is zero. Overflow, underflow, or loss of precision cannot occur.

Related actions: add(), sub(), mul(), div(), math(), sqrt(), random()

 

Compatibility:

FKiSS 3 mod(X,A,B)  or  mod(X,A,3) integer division, single letter variable names
FKiSS 5 mod(MyVariable,A,3.145) or mod(X,A,999999999999) integer, string, floating point division, long variable names

 

UltraKiss Options:

VariableCase variable names are case sensitive default value "false" (Compatibility option)

 

Example Code (FKiSS3): 

;@let(A,1) let(B,2) let(C,3)                         ; 
;@mod(X,A,B)                                         ; simple division
;@mod(X,C,-1)                                        ; literal constants

Example Code (FKiSS5): 

;@let(Apple,1) let(Pear,2) let(PI,3.1415927)         ; 
;@mod(TwoPie,PI,PI)                                  ; floating point division
;@mod(OnePie,TwoPie,-3.1415927)                      ; floating point constant
;@mod(BigPie,OnePie,-1234567890123456)               ; long constant
;@mod(X,4,2)                                         ; integer arithmetic
