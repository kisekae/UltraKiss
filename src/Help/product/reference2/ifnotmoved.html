ifnotmoved()

Syntax:     ifnotmoved (target,alarmname,duration)

The ifnotmoved() statement tests a target object or image cel to determine if it is positioned at its original page location from when the set was first initialized. If the target has not moved a timer is set for the specified alarm to fire after the specified time duration. Note that movement is context dependent on the current visible page, thus an ifnotmoved() command will not schedule the alarm on any page where the object or image cel has moved and will schedule the alarm on any page where the object or image cel has not moved.

If cels are moved the offset of the cel within the object group is changed. This can imply that the object has moved as the object location may change. This will occur if the cel is moved above or to the left of the current object location.

The target can be a literal object number or a cel name, or a variable that maintains a valid representation of an object or cel name. The ifmoved() command does not apply to cel groups.

The alarmname is an identifier for an alarm or a string variable that contains the name of an alarm. If the object has not moved from its initial location on the page then a timer is set for the specified alarm to fire after the specified time duration.

The duration is an integer number or a variable with an integer value that specifies the duration time in milliseconds.

An object initial position is specified in the page set specification in the configuration file. This command is context sensitive and applies to initial object positions on the current page.

Note also that it is possible to restore an object to its initial location using an FKiSS move command. The object's original x and y coordinates can be retrieved through the letinitx() and letinity() commands. If the object position is restored then the ifnotmoved() command considers that the object has not moved.

This command fails silently if the target or alarm does not exist.

 

FKiSS 5 Syntax:     ifnotmoved(target,alarmname,duration[, argument1, argument2, ...])

UltraKiss FKiSS5 introduces the ability to pass parameters to the alarm code module, similar to parameters on label modules. The optional parameter list specifies a set of variables or literal constants that can be referenced as local variables within the alarm code module. With FKiSS 5 the alarm name can also be specified through a variable that has a string value, providing that the variable name is not the same as an existing alarm name.

The optional argument list specifies a set of variable names that must be referenced as local variables within the alarm module.

Local variable names are prefixed with an '@' sign. Thus, the variable '@N' is a local variable declared and referenced within the module and is distinct from any global variable 'N'. Calling any module recursively creates new instances of all local variables distinct from any other local variable of the same name.

 

 

Related actions: ifmoved(), ifmapped(), iffixed(), move(), ifnotfixed(), ifnotmapped(), letinitx(), letinity()


Compatibility:

FKiSS 2 ifnotmoved(#5,12,1000) object comparisons, integer alarm identifiers
FKiSS 5 ifnotmoved(Target,NextAlarm,Time) or ifnotmoved("face.cel",12,T,Param1,Param2) variables for objects and cels, with alarm parameters

 

UltraKiss Options:

VariableCase variable names are case sensitive default value "false" (Compatibility option)
