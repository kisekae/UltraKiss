Sailor Moon Tutorial

Lesson 5. Correcting the Errors

 

  1. If you tested the events from exercise 4 you should notice that when switching back and forth from page 0 to page 1, sometimes the sailor costume is not visible.  This is because our event logic is not complete.  If you are continuing on from lesson 4 you can skip to step 3.  Otherwise, ensure that UltraKiss is showing the main logo screen.  Close any existing data set. If you want to ensure that your new KiSS set is created to be compatible with other viewers, press one of the viewer compatibility preset buttons on the toolbar.

     

  2. Use File-Open command to load the configuration file CNF4.cnf from the Tutorial/Sailor Moon directory or the directory you used in lesson 4.  This will load the Sailor Moon KiSS set created in exercise 4.

     

  3. For our set(1) event we quite properly made the sailor costume invisible but we forgot to make the school uniform visible.  Our set(1) event is not complete.  This time we will use the FKiSS Editor to edit our event. Use Tool-FKiSS Editor to open the editor.  Select the set(1) event in the tree and use File-Edit Event to open a text editor.  Change the complete set(1) event code to be as follows, using the correct object numbers from your data set:

    ;@set(1)
    ;@ unmap(#4) unmap(#5) unmap(#6) unmap(#7) unmap(#8)
    ;@ map(#10) map(#11) map(#12) map(#13)

    Note:  If you copy and paste these lines from the help documentation into your configuration file you must ensure that the semicolon and at-sign combination appears at the beginning of every line.  You can copy the FKiSS text by selecting the text and pressing Ctrl-C to copy the text to the clipboard, and Ctrl-V to paste the text into the text editor.

    Exit the text editor to apply your changes.

     

  4. Similarly, we should write a set(0) event to ensure that the sailor costume is visible whenever page 0 is shown.  We don't need to worry about the school uniform on page 0 as these objects do not apply to this page.  In the FKiSS Editor, use File-New Event. This will open a text editor that is initially empty. Code a set(0) event as shown, using the object numbers from your data set:

    ;@set(0)
    ;@ map(#4) map(#5) map(#6) map(#7) map(#8)

    Note:  If you copy and paste these lines from the help documentation into your configuration file you must ensure that the semicolon and at-sign combination appears at the beginning of every line.  You can copy the help text by selecting the text and pressing Ctrl-C to copy the text to the clipboard, and Ctrl-V to paste the text into the text editor.

     

  5. Exit the FKiSS Editor with File-Exit, or if you want to leave the FKiSS Editor active for future changes you can resume your set execution with Run-Program Resume or pressing the green dot on the toolbar. 

     

  6. Test your changes.  Switch back and forth from page 0 to page 1 and observe that the sailor costume shows properly.  If there is a problem use View-Active Configuration and verify that all set(0) and set(1) event logic is correct and references the actual object numbers in your configuration.

     

  7. You may also have noticed that the doll body is not permanently fixed in place.  If you try, you can pick up and move the doll from underneath the clothes.  This is usually not desired.  We fix this as follows.

    Place the mouse over any part of the doll not covered by clothing, such as the face or arms or legs.  Right click with the mouse and a popup menu for the doll object will appear.  Select Image Context - Properties.  A dialog window for the doll image will be shown.  A picture of the doll will be in the preview area of this dialog window.

    Press the small button beside the Lock Value setting in the attributes section.  Enter the value 99999.  Press the OK button to close the Properties dialog and then observe that the doll image can no longer be picked up and moved with the mouse.  The lock value has fixed the doll in place because the value 99999 exceeds the value set for a maximum fixed object.

     

  8. We can have fun with lock values.  For example, place the mouse over the school uniform skirt or fuku and right click.  Select Image Context-Properties and change the skirt lock value to 5.  This will make the skirt 'sticky', or slightly difficult to remove from the doll.  After closing the Properties dialog window try to move the skirt and observe the result.

    You should see the skirt move slightly and then snap back to its initial place. This is called a sticky object because it cannot be easily moved. If you keep trying to move the skirt eventually it will break free. At this time you are allowed to move the skirt anywhere. The skirt broke free on the fifth attempt because the lock value had been set to 5.

    You can restore the object sticky behaviour if you View-Reset the page.

     

  9. Use File-Save to save your work.  The corrected event code will replace the CNF4.cnf file previously loaded and the new object lock values will be saved in the configuration file.
