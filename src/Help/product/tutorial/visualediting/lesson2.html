Visual Editing Tutorial

Lesson 2. Understanding Copy and Paste

 

  1. This tutorial will show you how to copy and paste image cels and objects within and across pages in your KiSS set.  You will learn about ambiguous cels, images that are ambiguous because the same cel is used in more than one place on the same page. If you are continuing from lesson 1 you can skip to step 3. Otherwise, ensure that UltraKiss is showing the main logo screen. Close any existing set.

     

    This tutorial requires image files. If the necessary files cannot be found on your computer download the tutorial files using Help-Tutorial Files.

     

  2. Use File-Open  to load the configuration file CNF1.cnf from the Tutorial/Visual Editing directory or the directory you used in lesson 1.  This will load the Visual Editing set that you created in lesson 1.

     

    The set will open to page 0. Four buttons should show. Two are button components and two are cel images that look like buttons.

     

  3. Use Edit-Add Page to create new page 1. Our buttons will switch the set to new pages. UltraKiss will automatically switch to view this empty page. It will be black.

     

    Use View-Page Set and choose set 0 from the pop-up menu to return to our menu page.

     

  4. Next, right-click on the first component button with the label button1. A contexual selection popup window will show. Select Image Context and then select Event Wizard.

     

  5. The FKiSS Event Wizard dialog will show. The Event Source should be set to 'JavaCel n button4.button', the first component button, where 'n' is probably 3 but could be another number. Follow the instructions in the Wizard window. Choose the 'press' event from the Event selection box.

     

  6. The text area should now display the generated FKiSS code for a press event on the selected button cel. The object number '#n' from above is used. A default action statement to display a notify dialog is automatically generated for you.

     

  7. Overtype or replace the ;@notify action statement with ;@ changeset(1) statement. This statement causes a switch to page set 1. If your button is button4.button, your new event code should look as shown below:

     

    ;@press("button4.button")
    ;@ changeset(1)
    Note:  If you copy and paste these lines from the help documentation into your configuration file you must ensure that the semicolon and at-sign combination appears at the beginning of every line.  You can copy the help text by selecting the text and pressing Ctrl-C to copy the text to the clipboard. Use Ctrl-V to paste the text into the configuration file.

     

  8. Click OK to create the new event. If there are no syntax errors in the event code it will become part of the new event code for the set and you can actually try out the event. Go to the next step to continue.

     

    If there is a syntax error you will receive an error message and the Text Editor will be automatically invoked so that the error can be corrected.

     

    Note, if you happen to paste the ;@ changeset(1) statement on the same line as the ;@press() statement you will not see an error, but the button press will not work. The semicolon on the ;@ changeset(1) statement is being treated as a comment applied to the press() event. The changeset(1) statement is being ignored.

     

    Also, if you skipped Step 3 and did not create a new page 1, the button will not work. Ensure that page 1 actually exists. Use View-Page Set and verify that the pop-up menu shows both page 0 and 1.

     

    If you have problems close this set with File-Close and start again. There are other ways to correct FKiSS code using the FKiSS Editor tool, but that is the topic of another lesson.

     

  9. Press the button. Or use View-Page Set and choose set 1 from the pop-up menu. UltraKiss will switch to page 1 and show a black screen.

     

  10. Use Edit-Import Image to Selection and choose BALL.CEL from the Tutorials/Visual Editing directory. If prompted to select a palette for this cel choose BALL.KCF.

     

    The image file will be imported and the page color will change to blue. The page color changed to blue because the background color for a KiSS set is taken from the first color in the first multipalette in the first palette file in the set. Another tutorial will show you how to set and change palette colors.

     

    The ball image will be drawn with a green marquee box as it is a selected image cel. Do not move this ball.

     

    Use Edit-Copy to copy this image cel to the clipboard. Remember, this image cel should have a green marquee box as it is the selection set that is copied.

     

    Use Edit-Paste to paste a copy of this ball into the page. The selected marquee box will flash as the second ball is pasted over top of the first ball.

     

  11. Repeat the Edit-Paste seven more times. You can do this easily using the Ctrl-V shortcut key for the paste operation. Once again the selected marquee box will flash for each paste.

     

    Check that you have nine copies of the ball in the set. Use F11 or View-Active Configuration to open the text editor tool to see the current confiuration CNF file. Count the number of BALL.CEL entries in the object group. Close the text editor with File-Exit.

     

  12. Use Edit-Regroup to combine all nine selected image cels into one group object. Right-click on the background to remove all selections.

     

    Each time you pasted a copy of the balls into the set you created an ambiguous cel. Ambiguous cels all have the same name. The same image cel is used more than once in the KiSS set. The cels are ambiguous because FKiSS events or other action commands applied to the cel will apply to all copies of the same cel.

     

  13. Use View-Page Set and choose set 0 from the pop-up menu to return to our menu page.

     

    Select the object group with the Press Me buttons. Right-click over one of the buttons and choose Select. Both buttons should be encased in a blue marquee box.

     

    Use Edit-Copy to copy this object to the clipboard.

     

  14. Switch to page 1. Use View-Page Set and choose set 1 from the pop-up menu.

     

    Use Edit-Paste to paste a copy of the buttons into page 1. You can again use the Ctrl-V shortcut key for the paste operation.

     

    Move the button object to the upper right corner of the page. Then use Edit-Write Page to set the initial position for these objects on page 1. These buttons are also ambiguous cels.

     

  15. Use File-Save As to save your work.  Save the new KiSS set to the Tutorial/Visual Editing directory or a directory of your choice.  The configuration element name should be CNF2.cnf if you began this tutorial after first starting UltraKiss.  You can overwrite any existing element of the same name.
