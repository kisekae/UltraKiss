Visual Editing Tutorial

Lesson 4. Edit Undo and Redo

 

  1. This tutorial will explore the undo and redo operations that exist for most visual editing commands. If you are continuing from lesson 3 you can skip to step 3. Otherwise, ensure that UltraKiss is showing the main logo screen. Close any existing set.

     

    This tutorial requires image files. If the necessary files cannot be found on your computer download the tutorial files using Help-Tutorial Files.

     

  2. Use File-Open  to load the configuration file CNF3.cnf from the Tutorial/Visual Editing directory or the directory you used in lesson 1.  This will load the Visual Editing set that you created in lesson 3.

     

    The set will open to page 0. Four buttons should show. Two are button components and two are cel images that look like buttons. One of those buttons has been programmed with an event to switch to page 1 of your KiSS set.

     

  3. Use Edit-Add Page to create new page 2. UltraKiss will automatically switch to view this empty page. It will be the default color of the set palette.

     

    Use View-Page Set and choose set 0 from the pop-up menu to return to our menu page.

     

  4. Next, right-click on the first component button with the label button2. A contexual selection popup window will show. Select Image Context and then select Event Wizard.

     

  5. The FKiSS Event Wizard dialog will show. The Event Source should be set to 'JavaCel n button5.button', the first component button, where 'n' is probably 3 but could be another number. Follow the instructions in the Wizard window. Choose the 'press' event from the Event selection box.

     

  6. The text area should now display the generated FKiSS code for a press event on the selected button object. The object number '#n' from above is used. A default action statement to display a notify dialog is automatically generated for you.

     

  7. Overtype or replace the ;@notify action statement with ;@ changeset(2) statement. This statement causes a switch to page set 2. If your button is object 3, your new event code should look as shown below:

     

    ;@press("button5.button")
    ;@ changeset(2)
    Note:  If you copy and paste these lines from the help documentation into your configuration file you must ensure that the semicolon and at-sign combination appears at the beginning of every line.  You can copy the help text by selecting the text and pressing Ctrl-C to copy the text to the clipboard. Use Ctrl-V to paste the text into the configuration file.

     

  8. Click OK to create the new event. If there are no syntax errors in the event code it will become part of the new event code for the set and you can actually try out the event. Go to the next step to continue.

     

    If there is a syntax error you will receive an error message and the Text Editor will be automatically invoked so that the error can be corrected.

     

    Note, if you happen to paste the ;@ changeset(2) statement on the same line as the ;@press() statement you will not see an error, but the button press will not work. The semicolon on the ;@ changeset(2) statement is being treated as a comment applied to the press() event. The changeset(2) statement is being ignored.

     

    Also, if you skipped Step 3 and did not create a new page 2, the button will not work. Ensure that page 2 actually exists. Use View-Page Set and verify that the pop-up menu shows both page 0 and 1 and 2.

     

    If you have problems close this set with File-Close and start again. There are other ways to correct FKiSS code using the FKiSS Editor tool, but that is the topic of another lesson.

     

  9. Press the button. Or use View-Page Set and choose set 2 from the pop-up menu. UltraKiss will switch to page 2 and show an empty screen.

     

  10. Use View-Restart to reset the initial cel offsets in the configuration file. If you do not do this then the current offset values for all ambiguous cels will be written to the CNF file when the set is saved. A restart of the set will restore the initial offets that were first used.

     

    If you happen to inadvertently save the set with current offsets then when you reload the set your balls will appear as a circle. You will have to edit the configuration with View-Active Configuration and remove all %offset[] tags from all cel specifications.

     

    Use File-Save As to save your work.  Save the new KiSS set to the Tutorial/Visual Editing directory or a directory of your choice.  The configuration element name should be CNF4.cnf if you began this tutorial after first starting UltraKiss. You can overwrite any existing element of the same name.
