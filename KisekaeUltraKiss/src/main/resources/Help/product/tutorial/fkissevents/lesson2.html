Lesson 2. Editing FKiSS Events

 

  1. We are now going to edit the behavior of the simple button events we created in lesson 1.  If you are continuing from lesson 1 you can skip to step 3.  Otherwise, ensure that UltraKiss is showing the main logo screen.  Close any existing data set.

     

  2. Use File-Open  to load the configuration file CNF1.cnf from the Tutorial/FKiSS directory or the directory you used in lesson 1.  This will load the button set that you created in lesson 1.

     

  3. Our buttons will switch the set to new pages.  Use Edit-Add Page to create new page 1.  UltraKiss will automatically switch to view this empty page. 

     

  4. Next, use Edit-Import Image to New Object and select rei.jpg from the Tutorial/FKiSS directory.

    If you have any of the viewer compatibility presets this JPG image will be automatically converted to a cherry KiSS cel. Images are also converted to cels if the Options-Options-Import-Import all images as Cel files option is set.

    Position the image in the center of the screen.

     

  5. Repeat step 3 and step 4 to create new page 2.  This time, import ruri.jpg as the image file. 

     

  6. Next, use View-Page Set 0 or use the toolbar button to display page 0 with the buttons.  Right-click on the first button to display the context menu.  Choose the Object Context, then Edit Events.  This will activate the FKiSS Editor tool. Events can be specified for objects or cels. In the previous tutorial we set the event for the first button on the object.

    You could also have used Tools-FKiSS Editor to do this. In this case the FKiSS Editor will show all events for the set, whereas if you invoke the editor through the property dialog you will see only the events for the contextually selected object or image cel.

     

  7. Click on the 'press(#1)' event to highlight it.  Next, use File-Edit Event to open the Text Editor for the currently selected event.  You will see the event code from lesson 1.

     

  8. Overtype or replace the ;@notify action statement with ;@ changeset(1) statement.  This statement causes a switch to page set 1.  If your button is object 1, your new event code should look as shown below:

    ;@press(#1)
    ;@ changeset(1)
    Note:  If you copy and paste these lines from the help documentation into your configuration file you must ensure that the semicolon and at-sign combination appears at the beginning of every line.  You can copy the help text by selecting the text and pressing Ctrl-C to copy the text to the clipboard. Use Ctrl-V to paste the text into the configuration file.

     

  9. Exit the Text Editor and apply your changes.  If you wish you can exit the FKiSS Editor and test your changes.  Press the first button on page 0 and your set should automatically switch to page 1.

     

  10. Repeat steps 6 through 9 to edit the event code for the second button.  In the previous tutorial we set the event for the second button on the image cel. Use Image Context and then Edit Events.  This will activate the FKiSS Editor tool and show only the events for the cel.

    This time, replace the ;@notify action statement with ;@ changeset(2) statement to cause a switch to page set 2.  For button 2, your new event code should look as shown below:

    ;@catch("button2.cel")
    ;@ changeset(2)

    Note:  If you copy and paste these lines from the help documentation into your configuration file you must ensure that the semicolon and at-sign combination appears at the beginning of every line.  You can copy the help text by selecting the text and pressing Ctrl-C to copy the text to the clipboard. Use Ctrl-V to paste the text into the configuration file.

     

  11. Exit the Text Editor and apply your changes.  If you wish you can exit the FKiSS Editor and test your changes.  Press the second button on page 0 and your set should automatically switch to page 2.

     

  12. If you like, you can also define new press events on each of the images to easily return to page 0.  This is left as an exercise for you. 

     

  13. Use File-Save As to save your work.  Save the new KiSS set to the Tutorial/FKiSS directory or a directory of your choice.  Change the name to CNF2.cnf.  You can overwrite any existing element of the same name.
